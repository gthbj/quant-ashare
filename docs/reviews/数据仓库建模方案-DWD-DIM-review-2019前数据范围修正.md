## 修正结论

原先把“2019 前数据”理解成“DWD/DIM 全历史写入”过宽。当前阶段的正确口径是：

| 类别 | ODS 读取/接口参数 | DWD/DWS 写入 | 说明 |
|---|---|---|---|
| 财务/事件 | 从 `20170101` 起 | 财务/事件 DWD 写 2017+；DWS 样本写 2019+ | PIT、公告滞后、同比/基期需要 |
| 行情/估值/资金 | 写 2019+，构建时按最大滚动窗口多读 2018 buffer | 最终 DWD/DWS 只写 2019+ | `ret_1d`、MA/波动率等 warm-up |
| 维度/日历 | 取最新快照或全量事件历史 | 维度全量 / SCD2 全量 | `trade_cal`、`stock_basic`、`namechange` 天然承载历史 |

正式建设 2019 年以前样本/明细，是后续阶段，不进入当前 P0 范围。

## 对早先 review 的修正

- “DWD/DIM 初始写入 ODS 可用全历史”不作为项目决策；已由新决策取代。
- 早期 `stk_limit` 2007 前、`suspend_d` 1999 前覆盖缺口，不构成 P0 阻塞，因为行情最终只写 2019+，lookback buffer 读到 2018 即可。
- `daily_basic` 全跨度单次查询的外部文件问题，不作为当前 P0 估值 DWD 的阻塞项；仍保留“大范围回填分批跑”的工程建议。
- `dim_stock` / `trade_cal` / `namechange` 属于快照或全量历史维度，不等价于“按分区拉旧行情”。
