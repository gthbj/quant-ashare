## 1. 背景

财务 ODS 属于 B 类强 PIT 表：`partition_date == 报告期(end_date)`，不是公告日。同一只股票、同一报告期可能因为不同 `report_type`、公告修正和 `update_flag` 出现多条记录。

当前主方案已写明：

- 回测特征必须用 `ann_date_eff` / `visible_trade_date` 做 as-of join。
- 财务版本事实表保留历史公告版本，`_latest` 便捷表不能用于回测。
- 消费侧默认取合并报表 `report_type = '1'`。

OQ-003 尚未关闭，因为还需要 owner 确认：平台是否只支持默认合并报表口径，还是需要把母公司、单季、调整口径等作为可选维度保留并向下游暴露。

## 2. 问题定义

如果不显式建模财务口径，会产生两类风险：

1. **口径混用**：同一 `(sec_code, report_period)` 下合并报表、母公司报表或其他口径混在同一个 as-of 排序中，下游可能取到不可比的 ROE、利润率、现金流指标。
2. **未来扩展受限**：若 DWD 过早过滤掉非默认 `report_type`，后续要研究母公司口径、单季口径或口径差异因子时，需要重新回填财务明细表。

本文的核心取舍是：P0 DWS 默认只消费合并报表，DWD 层是否保留足够口径字段以支撑未来扩展。

## 3. 目标与非目标

### 3.1 目标

1. 明确 P0 默认财务口径：合并报表 `report_type = '1'`。
2. 规定财务 DWD 版本事实表对 `report_type` / `report_caliber` 的保留方式。
3. 规定 `dwd_fin_*_latest` 与 DWS 财务 as-of join 在多口径场景下的去重键、排序和 QA。
4. 防止 DWS / ADS 训练样本混用不同财务报表口径。
5. 给 owner 一个可审阅的决策面：P0 只落默认口径，P1 再开放多口径特征。

### 3.2 非目标

- 不改变已物化策略 1 价格量价 DWS/ADS。
- 不重建当前 `dwd_fin_indicator` 实表；若源表无 `report_type` 字段，不伪造多口径。
- 不定义所有 Tushare `report_type` 编码的业务解释，以落地前实际 ODS schema 和 Tushare 文档核对为准。
- 不把母公司口径、单季口径作为 P0 训练默认特征。
- 不解决 OQ-005 调度选型、OQ-010 策略默认参数或 OQ-011 lookback 输入问题。

## 4. 推荐决策

| 决策项 | 推荐口径 | 原因 |
|---|---|---|
| P0 默认消费口径 | `report_type = '1'`，合并报表 | A 股量化常用集团整体经营口径，可比性最好 |
| DWD 是否保留多口径 | 对 `income` / `balancesheet` / `cashflow` 等源表有 `report_type` 的表，版本事实表保留全部口径；默认消费视图/下游过滤合并口径 | 避免早期过滤导致未来回填 |
| DWS 是否物化多口径 | P0 只物化默认合并口径；多口径 DWS 延后到 P1+ | 保持首版特征宽表简单、主键不膨胀 |
| `fina_indicator` 特例 | 若 ODS schema 无 `report_type`，保持 source default，不新增伪造口径；输出 `report_caliber = 'source_default'` 仅作为元数据标识 | 不制造无法追溯的业务含义 |
| 单季指标 | 优先使用已存在的 `q_*` 指标；三大报表单季派生作为 P1 财务表实现内容 | 避免把累计口径和单季口径混在一个字段里 |

owner 若采纳本文推荐，可将 OQ-003 关闭为：**P0 默认合并报表；DWD 保留源口径字段，DWS 默认只消费合并口径，多口径特征后续另建/扩展。**

## 5. 数据模型要求

### 5.1 DWD 版本事实表

适用表：

- `dwd_fin_income`
- `dwd_fin_balancesheet`
- `dwd_fin_cashflow`
- 后续其他源表明确提供 `report_type` 的财务表

必备字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `sec_code` | STRING | 证券主键 |
| `report_period` | DATE | 报告期，来自 `end_date` |
| `ann_date_eff` | DATE | 可见日，三大报表用 `COALESCE(f_ann_date, ann_date)` |
| `visible_trade_date` | DATE | 公告后第一个可建仓交易日 |
| `report_type` | STRING | 源表报表类型编码，原样保留 |
| `report_caliber` | STRING | 规范化口径，首版至少支持 `consolidated`、`non_consolidated`、`other`、`unknown` |
| `is_default_report_caliber` | BOOL | 是否为 P0 默认消费口径，首版等价于 `report_type = '1'` |
| `update_flag` | STRING | Tushare 更新标志，优先保留修正版 |
| `source_system` | STRING | 来源系统 |
| `source_partition_date` | STRING | ODS 分区，仍是报告期分区 |
| `ingested_at` | TIMESTAMP | 摄入时间 |

版本事实表不按 `report_type` 预过滤。去重键必须包含 `report_type`：

```sql
PARTITION BY sec_code, report_period, report_type, ann_date_eff, update_flag
ORDER BY ingested_at DESC, source_partition_date DESC
```

### 5.2 最新便捷表

每张财务版本事实表可配套一个 `_latest` 便捷表。`_latest` 仍不得用于 PIT 回测。

默认便捷表建议只产出合并报表口径：

```sql
WHERE is_default_report_caliber = TRUE
QUALIFY ROW_NUMBER() OVER (
  PARTITION BY sec_code, report_period
  ORDER BY update_flag DESC, ann_date_eff DESC, ingested_at DESC, source_partition_date DESC
) = 1
```

若未来需要多口径 latest，另建 `*_latest_by_caliber` 或把主键扩展为：

```text
(sec_code, report_period, report_caliber)
```

不得在同一个 latest 表里既声明唯一键 `(sec_code, report_period)`，又保留多条口径。

### 5.3 `dwd_fin_indicator` 特例

当前 P0 已物化 `dwd_fin_indicator`，用于 `fina_indicator` 已计算财务比率。该表已按 PIT 版本事实表设计，且 `fina_indicator` 已实测无 `f_ann_date`，可见日只能用 `ann_date`。

后续处理规则：

- 若 ODS `fina_indicator` 无 `report_type` 字段，维持现有版本事实表，不为历史数据伪造 `report_type='1'`。
- 可在后续字段扩展中增加 `report_caliber = 'source_default'`、`is_default_report_caliber = TRUE`，用于 DWS 统一字段命名。
- 若上游未来补出 `report_type`，再按 §5.1 的方式纳入版本键，并单独提迁移 PR。

## 6. DWS 消费要求

### 6.1 `dws_stock_feature_fin_daily`

P0 财务 DWS 只消费默认口径。对三大报表或后续带 `report_type` 的财务表，默认口径过滤必须放在预过滤 CTE / 子查询中，或放在 `LEFT JOIN ... ON` 条件内；不得写到 `WHERE` 子句，否则会把 `LEFT JOIN` 退化为 inner join 并丢掉暂无财务记录的股票日期。

推荐写法：

```sql
LEFT JOIN (
  SELECT *
  FROM ashare_dwd.dwd_fin_income
  WHERE is_default_report_caliber = TRUE
) AS fin
  ON fin.sec_code = u.sec_code
 AND fin.visible_trade_date <= u.trade_date
```

`dwd_fin_indicator` 若仍无 `report_type` 字段，可按 `report_caliber='source_default'` 统一输出元数据；在字段扩展完成前，继续只用 `visible_trade_date <= trade_date` 做 PIT 截点。

as-of 排序必须在同一默认口径内完成，并保持和主方案 §7.3 一致：先按 `visible_trade_date <= trade_date` 保留当时可见版本，再按报告期和公告生效时间取“当时已可见的最新版本”。`update_flag` 只作为同公告时间下的修正版 tie-breaker，不得优先于 `ann_date_eff`：

```sql
QUALIFY ROW_NUMBER() OVER (
  PARTITION BY u.sec_code, u.trade_date
  ORDER BY fin.report_period DESC,
           fin.ann_date_eff DESC,
           fin.update_flag DESC,
           fin.ingested_at DESC,
           fin.source_partition_date DESC
) = 1
```

输出字段至少包含：

| 字段 | 说明 |
|---|---|
| `report_period` | 被 as-of 选中的报告期 |
| `ann_date_eff` | 被选中版本的公告生效日 |
| `visible_trade_date` | 被选中版本的可交易可见日 |
| `report_caliber` | 默认 `consolidated` 或 `source_default` |
| `is_default_report_caliber` | 默认 TRUE |
| `report_age_days` | `trade_date - report_period` |
| `fin_report_lag_days` | `visible_trade_date - report_period` |

### 6.2 多口径 DWS 扩展

若 owner 后续确认需要同时训练多口径特征，不能在现有 `(sec_code, trade_date, feature_version)` 主键下直接输出多行。

可选方案：

1. **宽字段方案**：在同一行增加 `roe_consolidated`、`roe_parent` 等字段。适合少量核心指标对照。
2. **长表方案**：新增 `dws_stock_feature_fin_caliber_daily`，主键为 `(sec_code, trade_date, feature_version, report_caliber)`。适合系统性研究多口径。

P0 不实现上述扩展，只为 DWD 留足口径字段。

## 7. 功能需求

### FR-1 DWD 保留源报表口径

带 `report_type` 的财务 DWD 必须原样保留 `report_type`，并派生 `report_caliber` 与 `is_default_report_caliber`。

验收：

- DWD schema 存在 `report_type`、`report_caliber`、`is_default_report_caliber`。
- 同一 `(sec_code, report_period, report_type, ann_date_eff, update_flag)` 去重后唯一。
- `report_type IS NULL` 的行必须落为 `report_caliber='unknown'`，不得默认当作合并报表。

### FR-2 默认 latest 只保留合并口径

默认 `_latest` 便捷表只输出 `is_default_report_caliber = TRUE` 的记录，且 `(sec_code, report_period)` 唯一。

验收：

- `COUNTIF(is_default_report_caliber IS NOT TRUE) = 0`。
- `(sec_code, report_period)` 无重复。
- 排序优先级为 `update_flag DESC, ann_date_eff DESC, ingested_at DESC, source_partition_date DESC`。

### FR-3 DWS 财务特征不混口径

P0 `dws_stock_feature_fin_daily` 只输出默认口径，且明示口径字段。

验收：

- `COUNTIF(is_default_report_caliber IS NOT TRUE) = 0`。
- `(sec_code, trade_date, feature_version)` 唯一。
- 随机抽样 as-of 记录满足 `visible_trade_date <= trade_date`。
- 不读取 `_latest` 表构造回测特征。

### FR-4 多口径扩展必须显式改键或改字段

若后续 PR 引入母公司或其他口径特征，必须选择 §6.2 的宽字段或长表方案之一，并在 PRD / 设计文档中说明主键变化。

验收：

- 不允许在现有 P0 财务 DWS 主键下因为多口径输出重复行。
- 多口径字段名称或 `report_caliber` 取值必须可追溯。

## 8. QA 断言

新增财务表落地时至少补充以下 QA。下列示例用 `dwd_fin_income`，`dwd_fin_balancesheet`、`dwd_fin_cashflow` 必须套用同一组断言。

```sql
-- DWD 必备口径字段存在
SELECT COUNT(*) AS missing_required_column_cnt
FROM UNNEST(['report_type', 'report_caliber', 'is_default_report_caliber']) AS required_column
LEFT JOIN `data-aquarium.ashare_dwd.INFORMATION_SCHEMA.COLUMNS` AS c
  ON c.table_name = 'dwd_fin_income'
 AND c.column_name = required_column
WHERE c.column_name IS NULL;

-- DWD 版本事实键唯一
SELECT COUNT(*) AS dup_cnt
FROM (
  SELECT
    sec_code,
    report_period,
    report_type,
    ann_date_eff,
    update_flag,
    COUNT(*) AS c
  FROM `data-aquarium.ashare_dwd.dwd_fin_income`
  GROUP BY 1, 2, 3, 4, 5
  HAVING c > 1
);

-- NULL report_type 不得被当成默认合并口径
SELECT COUNT(*) AS bad_unknown_cnt
FROM `data-aquarium.ashare_dwd.dwd_fin_income`
WHERE report_type IS NULL
  AND (report_caliber IS DISTINCT FROM 'unknown'
       OR is_default_report_caliber IS TRUE);

-- 默认口径映射必须和 report_type='1' 一致
SELECT COUNT(*) AS bad_default_mapping_cnt
FROM `data-aquarium.ashare_dwd.dwd_fin_income`
WHERE is_default_report_caliber IS DISTINCT FROM COALESCE(report_type = '1', FALSE);

-- 默认口径 latest 唯一
SELECT COUNT(*) AS dup_cnt
FROM (
  SELECT sec_code, report_period, COUNT(*) AS c
  FROM `data-aquarium.ashare_dwd.dwd_fin_income_latest`
  GROUP BY 1, 2
  HAVING c > 1
);

-- 默认口径 latest 不混入非默认口径
SELECT COUNT(*) AS non_default_cnt
FROM `data-aquarium.ashare_dwd.dwd_fin_income_latest`
WHERE is_default_report_caliber IS NOT TRUE;

-- DWS 财务特征 PIT 防线
SELECT COUNT(*) AS leak_cnt
FROM `data-aquarium.ashare_dws.dws_stock_feature_fin_daily`
WHERE visible_trade_date > trade_date;

-- DWS 财务特征主键唯一
SELECT COUNT(*) AS dup_cnt
FROM (
  SELECT sec_code, trade_date, feature_version, COUNT(*) AS c
  FROM `data-aquarium.ashare_dws.dws_stock_feature_fin_daily`
  GROUP BY 1, 2, 3
  HAVING c > 1
);
```

对 `fina_indicator` 保持现有 QA，并在未来字段扩展时增加：

- `report_caliber IN ('source_default')` 或空值迁移完成后不为空。
- `is_default_report_caliber = TRUE`。

**关键实现决策（落地时确认）**：

- **实测口径**：当前 ODS `income`/`balancesheet`/`cashflow` 仅含 `report_type='1'`（合并报表，分 `comp_type` 1/2/3/4/7）。`report_type>'1'` 的 `report_caliber` 映射（consolidated/non_consolidated/other）作为前向兼容写入 CASE，但当前数据不触发。回答 §11.2：当前唯一默认口径就是 `report_type='1'`。
- **可见日**：三大报表用 `ann_date_eff = COALESCE(f_ann_date, ann_date)`（区别于 `fina_indicator` 仅 `ann_date`）；`visible_trade_date` = 公告生效日之后第一个 SSE 开市日。
- **DWS 口径契约 vs 可用性**：`dws_stock_feature_fin_daily.report_caliber`/`is_default_report_caliber` 表示“本表三大报表特征只来自默认合并口径”的消费契约（恒 `consolidated`/`TRUE`），各来源是否真有数据由 `has_fin_*` 掩码和 `*_report_period` 标识；行数与 universe 完全一致（LEFT JOIN 未退化），满足 §6.1“不丢暂无财报股票日期”。
- **as-of 扇出约束（P0 简化）**：as-of 限制 `visible_trade_date ∈ [trade_date - 900d, trade_date]`；超 900 日（≈2.5 年）未更新财报视为缺失。正常季度披露的股票最新财报远在窗口内，不受影响。
- **单季派生**：按 §4 推荐延后到 P1；P0 直接用 `fina_indicator` 现成 `q_*`，三大报表绝对值保留累计/YTD 口径。
