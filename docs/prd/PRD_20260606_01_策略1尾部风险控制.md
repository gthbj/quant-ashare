# PRD: 策略 1 尾部风险控制与最大回撤诊断

> 状态：草案，待 owner review。
> 范围声明：本文定义策略 1 在 Cloud Run Python 执行路线下的尾部风险诊断、个股风险过滤和市场状态风控。P0 只新增固定诊断，不改变模型、选股、交易或回测结果；P1/P2 才允许作为独立实验改变候选池或仓位动作。
> 关联：`docs/prd/PRD_20260605_04_策略1CloudRunPython模型基线搜索.md`、`docs/prd/PRD_20260604_04_策略1CloudRun训练回测.md`、`docs/prd/PRD_20260605_02_策略1CloudRun轻量Task并发.md`、`docs/prd/PRD_20260604_01_策略1LedgerV1交易执行语义.md`、`docs/prd/PRD_20260602_04_策略1模型质量诊断.md`、`.agent/memory/OPEN_QUESTIONS.md` OQ-010。

## 1. 背景

PRD04 Cloud Run Python baseline search 已完成两轮失败事实：

1. sklearn native LogisticRegression 首轮 Top5 全部 rejected。
2. LightGBM binary wave 2 Top5 全部 rejected。
3. LightGBM regression wave 3 已完成 12 个候选、Top5 完整回测 / 报告 / 诊断和 `19` QA；Top5 仍全部 rejected。

Wave 3 regression 的关键特征不是“完全没有信号”。Top5 在 2025 test 期存在正向 RankIC 和部分正超额证据，但全部触发 `max_drawdown < -25%` 风险门，且多数候选在 2026 final holdout 跑输中证1000超过 5pct。

对 wave 3 Top5 的只读分析显示，最大回撤集中发生在 `2024-01` 至 `2024-02` 小盘股流动性冲击窗口：

1. Top5 最大回撤谷底均落在 `2024-02-07`。
2. 策略最大回撤约 `-32%` 至 `-37%`，同期中证1000回撤约 `-13%` 至 `-15%`。
3. `2024-02-05` 每个候选持仓 30 只，其中 26 至 28 只接近跌停，跌停 / 接近跌停仓位权重约 `86%` 至 `93%`。
4. 当天交易成本只有几十元量级，不能解释接近 `-10%` 的单日净值损失。
5. 信号日 `2024-02-02` 选中的股票没有 ST / universe / tradability 泄漏，但明显偏向近期大跌、高波动、高换手、小市值股票。

这说明当前问题不是单纯模型库选择问题，也不是 Ledger v1 成交语义 bug，而是策略缺少固定尾部风险诊断和显式风险控制。继续盲目扩模型族，会继续把“信号有效但尾部风险不可接受”的候选推进到昂贵的 Top5 回测阶段。

本文定义一条先诊断、再过滤、最后市场状态风控的方案。

## 2. 目标

### 2.1 P0 目标：最大回撤固定诊断

1. 对每个完整回测自动输出最大回撤区间。
2. 输出最大回撤窗口内的持仓贡献。
3. 输出回撤窗口内的跌停 / 一字板 / 不可卖风险仓位占比。
4. 输出回撤窗口前关键信号日和调仓日的选股画像：市值、波动、近期收益、20 日回撤、换手、跌停历史、行业和板块。
5. 将诊断 artifact 写入 GCS / 本地镜像，并回写 ADS summary / comparison artifact，供 PRD04 后续模型波次和 owner review 复用。
6. P0 不改变任何策略结果，只解释“为什么最大回撤发生”。

### 2.2 P1 目标：个股硬风险过滤实验

1. 在 prediction → candidate selection 之间新增可配置风险过滤 profile。
2. 首轮验证以下风险过滤是否能降低最大回撤，同时不过度牺牲 valid/test 信号和收益：
   - 近 20 日累计收益过差；
   - 近 20 日相对高点回撤过深；
   - 近 20 日跌停 / 一字板次数过多；
   - 市值过小；
   - 波动率或换手处于极端高位。
3. 所有过滤必须基于信号日 `t` 已可见特征，不得使用 `t+1` 或未来行情。
4. P1 只作为独立实验，不直接改现有 Cloud Run Python baseline search 默认口径。

### 2.3 P2 目标：市场状态风控

1. 引入市场层面的 risk-off 判断，识别小盘崩盘、跌停扩散和市场宽度恶化。
2. 在 risk-off 状态下对买入行为或目标总仓位做显式控制。
3. P2 必须依赖可审计的市场状态数据表或 artifact，优先推动 `dws_market_state_daily`。
4. P2 需单独验收，不与 P1 阈值调参混在同一次 acceptance 中。

## 3. 非目标

1. 不新增或扩展 BQML / `sql/ml/strategy1` SQL runner 策略执行路线。
2. 不改变 ODS / DIM / DWD / DWS 数仓事实口径。
3. 不改变 Ledger v1 交易执行语义。
4. 不改变首个基线股票池定义：仅沪深主板，不含北交所、创业板、科创板。
5. 不改变成本 profile：佣金万一免五、卖出印花税 5 bps、买 / 卖滑点各 5 bps。
6. 不把 P0 诊断结果直接当作交易规则。
7. 不引入新闻、公告、研报或外部文本解释最大回撤；没有结构化证据时必须写“当前证据不足”。
8. 不用 2025 test 或 2026 final holdout 反复搜索风险阈值。阈值搜索必须有固定候选集合和 test reuse 记录。

## 4. 术语

| 术语 | 说明 |
|---|---|
| `ret_20d` | 当前后复权收盘价相对 20 个交易日前后复权收盘价的累计收益率，公式为 `close_hfq / close_hfq_lag_20d - 1`。 |
| `drawdown_20d` | 当前后复权收盘价相对近 20 个交易日最高后复权收盘价的回撤，公式为 `close_hfq / close_hfq_max_20d - 1`。 |
| 最大回撤窗口 | 从某个历史 NAV 高点到后续 NAV 谷底之间的窗口。 |
| 持仓贡献 | 按日持仓权重与当日个股收益近似计算的组合收益贡献；若后续实现精确 ledger PnL，需同时输出近似贡献和精确贡献口径。 |
| 跌停仓位占比 | 回撤窗口内处于跌停、接近跌停或一字跌停状态的持仓权重占 NAV 或 gross exposure 的比例。 |
| 选股画像 | 在信号日 / 调仓日，对选中股票相对全预测池或 universe 的市值、波动、收益、回撤、换手和行业分布做横截面对比。 |
| risk-off | 市场状态进入小盘股 / 高波动股票明显不利的防守状态。 |

## 6. 功能需求

### FR-TAIL-1 运行身份冻结

每次诊断必须写入 `tail_risk_summary.json`，至少包含：

```json
{
  "tail_risk_diagnosis_version": "strategy1_tail_risk_v1",
  "strategy_id": "ml_pv_clf_v0",
  "search_id": "...",
  "run_id": "...",
  "backtest_id": "...",
  "benchmark_sec_code": "000852.SH",
  "ledger_version": "ledger_exec_v1",
  "tail_risk_profile_id": "diagnostic_only",
  "generated_at": "..."
}
```

如果同一 `backtest_id` 找不到唯一 summary 或 NAV 序列，P0 诊断必须失败。

### FR-TAIL-2 最大回撤窗口

对每个 `backtest_id` 计算：

1. 全区间最大回撤。
2. 前 K 个回撤事件，默认 K=5。
3. 每个事件的：
   - `peak_date`
   - `trough_date`
   - `trading_days`
   - `peak_nav`
   - `trough_nav`
   - `drawdown_pct`
   - 同期 benchmark return
   - 同期 excess return
   - 窗口内最差单日策略收益
   - 窗口内最差单日相对 benchmark 超额

输出：

```text
tail_risk/max_drawdown_windows.csv
tail_risk/max_drawdown_windows.json
```

### FR-TAIL-3 回撤窗口持仓贡献

对每个最大回撤窗口，按日输出持仓贡献：

```text
approx_daily_contribution = position_weight_start_or_eod * stock_ret_1d
```

P0 可先使用持仓日权重 × DWD 个股 `ret_1d` 做近似贡献。若实现可拿到 ledger 精确成本 / 成交 / mark-to-market PnL，则必须额外输出精确口径，不得覆盖近似口径。

聚合维度：

1. `sec_code`
2. 股票名称
3. 板块
4. 行业
5. 信号日是否入选 TopN
6. 是否所有候选共同持有
7. 回撤窗口累计贡献
8. 最差单日贡献
9. 回撤窗口平均权重
10. 回撤窗口持有天数

输出：

```text
tail_risk/drawdown_position_contribution.csv
tail_risk/drawdown_industry_contribution.csv
tail_risk/drawdown_board_contribution.csv
```

### FR-TAIL-4 跌停 / 流动性风险仓位

对最大回撤窗口内每个交易日输出：

1. 持仓只数。
2. 跌停或接近跌停只数。
3. 一字板只数。
4. `can_sell_open=FALSE` 或等价不可卖标记只数。
5. 跌停 / 接近跌停仓位权重。
6. 一字板仓位权重。
7. 不可卖仓位权重。
8. 当日策略收益、benchmark 收益和超额收益。

“接近跌停”的默认定义：

```text
ret_1d <= -0.095
```

若 DWD 已有 `is_limit_down`，则优先使用源标记；`ret_1d <= -0.095` 只作为补充量级标记。实现必须同时输出命中来源，避免把跌幅阈值误写成严格交易所跌停判定。

输出：

```text
tail_risk/limit_down_exposure_daily.csv
```

### FR-TAIL-5 选股画像

对每个最大回撤窗口，定位窗口前最近一次或多次调仓信号日，输出选中股票相对预测池 / universe 的画像。

P0 至少覆盖以下字段：

| 类别 | 字段 |
|---|---|
| 市值 | `total_mv_cny`、`circ_mv_cny`、`log_total_mv`、`log_circ_mv` |
| 近期收益 | `ret_5d`、`ret_20d`、`ret_60d` |
| 风险 / 波动 | `drawdown_20d`、`vol_20d`、`vol_60d`、`hl_range_20d` |
| 流动性 / 换手 | `amount_ma20_cny`、`turnover_rate_ma20`、`volume_ratio` |
| 跌停历史 | `limit_down_days_20d`、`one_word_limit_days_20d` |
| 样本标记 | `is_st`、`in_universe_default`、`is_tradable`、`has_full_history_60d` |
| 结构分布 | 板块、行业、上市年龄 |

对每个字段输出：

1. selected 中位数 / 均值 / p10 / p90。
2. prediction pool 或 universe 中位数 / 均值 / p10 / p90。
3. selected 与 universe 的差值或倍数。
4. 极端阈值命中数量，例如 `ret_20d < -30%`、`drawdown_20d < -30%`。

输出：

```text
tail_risk/selection_profile_by_signal_date.csv
tail_risk/selection_profile_summary.json
tail_risk/risky_selected_names.csv
```

### FR-TAIL-6 多候选重叠与共同踩踏

当输入 `search_id` 时，P0 必须支持 TopK 候选横向诊断：

1. 每个信号日 TopK 候选选股两两重叠数。
2. 被 3 个、4 个、5 个候选共同选中的股票。
3. 共同持仓股票在最大回撤窗口内的累计贡献。
4. 候选之间最大回撤窗口是否重合。

输出：

```text
tail_risk/candidate_overlap_by_signal_date.csv
tail_risk/common_crash_names.csv
tail_risk/search_tail_risk_summary.csv
```

### FR-TAIL-7 P1 个股硬风险过滤 profile

P1 新增可配置 profile：

```text
tail_risk_profile_id = individual_risk_guard_v0
```

首轮默认候选阈值：

| 规则 | 默认值 | 动作 |
|---|---:|---|
| `ret_20d < -30%` | TRUE | 排除买入候选 |
| `drawdown_20d < -30%` | TRUE | 排除买入候选 |
| `limit_down_days_20d >= 2` | TRUE | 排除买入候选 |
| `one_word_limit_days_20d >= 1` | TRUE | 排除买入候选 |
| `total_mv_cny < 30e8` | TRUE | 排除买入候选 |
| `circ_mv_cny < 20e8` | TRUE | 排除买入候选 |
| `vol_20d` 位于当日 prediction pool p95 以上 | TRUE | 标记；首轮不默认硬排除 |
| `turnover_rate_ma20` 位于当日 prediction pool p98 以上 | TRUE | 标记；首轮不默认硬排除 |

实施规则：

1. 过滤发生在 prediction pool 排名后、候选 TopN 生成前，或等价地在 candidate selection 查询中作为 eligible 条件。
2. 过滤只影响新的买入候选，不强制卖出已有持仓；已有持仓是否卖出仍由 Ledger v1 目标组合和 pending sell 语义决定。
3. 如果过滤后可选股票不足目标持股数，则实际持股数可以低于目标，剩余资金保留现金；不得用被过滤股票候补填满。
4. 每个调仓日必须输出被每条规则排除的数量、交集数量和最终 eligible 数量。
5. P1 回测必须与 `diagnostic_only` profile 做 A/B，对比收益、最大回撤、回撤窗口、换手和候选覆盖率。

输出：

```text
tail_risk/risk_filter_funnel_daily.csv
tail_risk/risk_filter_excluded_names.csv
```

### FR-TAIL-8 P2 市场状态风控

P2 需要市场状态表或 artifact。推荐优先新增 / 复用：

```text
ashare_dws.dws_market_state_daily
```

P2 首轮 risk-off 指标候选：

| 指标 | 示例 |
|---|---|
| 小盘指数趋势 | 中证1000 / 中证2000 / 国证2000 `ret_5d`、`ret_20d`、`drawdown_20d` |
| 市场宽度 | 主板上涨家数占比、创 20 日新低占比 |
| 跌停扩散 | 主板跌停家数、接近跌停家数、跌停市值权重 |
| 流动性冲击 | 主板成交额缩量、低成交股票占比 |
| 波动状态 | 指数 20 日波动率、横截面波动率 |

P2 v0 推荐动作：

```text
market_risk_action = skip_new_buys
```

语义：

1. risk-off 信号在信号日 `t` 收盘后形成。
2. `t+1` 执行时允许卖出和 pending sell 继续处理。
3. risk-off 状态下不新增买入，目标外旧仓按既有卖出规则处理。
4. 不做日内止损，不在非调仓日主动清仓。
5. 恢复 risk-on 后，下一次调仓日恢复正常买入。

更激进动作如 `target_gross_scale=0.5` 或 `cash_mode` 需另行 owner 确认后再实现。

### FR-TAIL-9 报告与 comparison 集成

P0 诊断必须接入 Cloud Run Python search comparison 输出：

1. comparison markdown 中显示 TopK 每个候选最大回撤、最大回撤窗口、同期 benchmark 回撤、跌停仓位峰值。
2. 单候选报告中新增“最大回撤诊断”章节。
3. GCS artifact 路径沿用：

```text

  search_id=<search_id>/
  run_id=<run_id>/
  backtest_id=<backtest_id>/
  tail_risk/
```

4. 本地镜像路径保持同结构。

## 7. 验收标准

### AC-TAIL-P0 诊断验收

P0 通过条件：

1. 任意给定 `run_id` + `backtest_id` 可生成完整 `tail_risk/` artifact。
2. `max_drawdown_windows.csv` 至少包含全区间最大回撤。
3. 最大回撤 `drawdown_pct` 与 `ads_backtest_performance_summary.max_drawdown` 误差不超过 `1e-6`。
4. `drawdown_position_contribution.csv` 覆盖最大回撤窗口内所有非零持仓股票。
5. `limit_down_exposure_daily.csv` 覆盖最大回撤窗口所有交易日。
6. `selection_profile_by_signal_date.csv` 至少覆盖最大回撤窗口前最近一次调仓信号日。
7. comparison artifact 能展示 TopK 候选最大回撤风险摘要。
8. P0 不改变 ADS backtest 结果、model registry、prediction、candidate、portfolio、order 或 NAV。

### AC-TAIL-P1 个股风险过滤验收

P1 通过条件：

1. `individual_risk_guard_v0` profile 参数化、可开关、可追溯。
2. 所有过滤字段均来自信号日已可见特征。
3. 每个调仓日输出过滤漏斗，且过滤后 candidate 生成逻辑无未来泄露。
4. 与 `diagnostic_only` A/B 对比中，最大回撤、收益、Sharpe、换手和覆盖率均有结构化结果。
5. 若 P1 想成为默认 profile，必须满足：
   - max drawdown 不低于共享契约阈值；
   - 2025 test 不跑输中证1000；
   - 2026 final_holdout 不触发明显坏结果 veto；
   - 不能只因为 `2024-02` 单一窗口改善就 accepted。

### AC-TAIL-P2 市场状态风控验收

P2 通过条件：

1. `dws_market_state_daily` 或等价 artifact 已实现并通过 PIT / 覆盖率 QA。
2. risk-off 判定只使用信号日已可见市场信息。
3. `skip_new_buys` 动作在订单和报告中可解释。
4. 所有 risk-off 日期、触发指标和动作均写入 artifact。
5. 与 P0/P1 baseline A/B 对比中，不能出现未解释的成交语义变化。

## 9. 实施顺序

## 11. 风险与边界

1. P1 阈值如果直接按 `2024-02` 反推，存在过拟合到单一崩盘窗口的风险。
2. 市值过滤可能明显改变策略风格，使其不再是原本的小盘 / 中小盘暴露策略。
3. 跌停历史过滤会降低踩踏风险，但也可能错过极端反弹机会。
4. P2 市场状态风控如果过于敏感，可能在反弹前减少买入，导致收益下降。
5. P0 持仓贡献如果先用权重 × 收益率近似，不能完全替代精确 ledger PnL；报告必须标明贡献口径。
6. 本文不解决“模型是否需要新特征 / 新标签”的全部问题，只先把尾部风险证据固定下来。
