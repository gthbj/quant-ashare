## 1. 背景

策略 1 BigQuery ML runner 已端到端跑通，报告 GCS uploaded 模式也已验收。当前问题不再是管线能否跑，而是模型质量明显不够：

- 当前基线 run：`run_id=s1_bqml_20260601_01`，`backtest_id=bt_s1_bqml_20260601_01`。
- 当前模型：BigQuery ML `LOGISTIC_REG`，5 个 `L1_REG/L2_REG` 候选，按 2024 valid RankIC 选型。
- 已知表现：valid RankIC 约为负、AUC 约 0.50，2025 回测 NAV 收于约 0.02。
- 现有中文报告可以解释“亏在哪里”，但还不足以回答“为什么模型信号差，以及下一步应该改标签、改特征、改选股、还是改组合参数”。

因此需要先做一轮结构化诊断，把策略差的根因拆清楚，再进入 OQ-010 的模型质量和参数迭代。否则容易根据 2025 单次回测结果直接调参，形成过拟合。

## 2. 目标

1. 产出可复核的模型质量诊断证据，覆盖 signal、label、sample/universe、candidate、portfolio、cost、market/style。
2. 判断当前差表现主要来自哪类原因：
   - 信号方向反了；
   - 信号没有信息量；
   - 标签定义与交易目标不匹配；
   - 训练 / 预测样本或股票池过滤口径有问题；
   - 组合参数过于集中或换手成本过高；
   - 市场风格 / 小盘 beta / 流动性暴露导致；
   - 执行和报告口径之外的数据质量问题。
3. 给下一轮实验设门槛：只有诊断证据支持，才进入反向分数、标签重做、参数网格、特征扩展等实现。
4. 保持可复现：所有结论来自 BigQuery ADS/DWS/DWD 和报告 artifact，不依赖口头判断。

## 3. 非目标

- 不在本 PRD 中修改 BQML 模型类型、特征列、标签列、股票池、持股数、权重上限或调仓频率。
- 不因 2025 test 回测差就直接采用反向分数；反向分数只能作为诊断结论后的独立实验。
- 不引入外部新闻、公告、研报文本解释亏损原因；没有外部证据时必须写“当前证据不足”。
- 不替代 `PRD_20260602_03_策略1中文报告归因分析.md` 的交易/亏损报告；本文聚焦模型质量和下一步实验决策。
- 不新增长期调度系统；P0 只要求可手动运行、可验收。

## 5. 关键决策

### 5.1 诊断优先于调参

P0 先实现诊断，不直接修改 runner 的模型、标签、选股、持仓或成本。

后续任何模型质量改动至少要引用本诊断的结论，例如：

- valid 与 test 都显示稳定反向信号，才允许做“反向分数”实验。
- score bucket 没有单调性且 RankIC 接近 0，优先改特征 / 标签，而不是调组合参数。
- score 有信息量但回测差，优先查组合集中度、换手成本、不可交易和风格暴露。

### 5.2 valid 是模型选择依据，test 是样本外验收

不能用 2025 test 反复挑参数。诊断可以在 test 上定位问题，但下一轮改动必须重新定义实验矩阵，并保留 train / valid / test 切分。

### 5.3 主基准保持中证 1000

诊断中的超额收益、beta、风格偏离和亏损归因主基准仍为中证 1000 `000852.SH`。沪深 300 `000300.SH` 保留为展示对比，不替代主评估基准。

### 5.4 诊断 artifact 是事实来源

面向 owner 的结论可以是中文，但必须能回溯到结构化 artifact。AI 分析只能基于诊断 artifact，不得补充未提供的数据源。

## 6. 诊断问题树

| 类别 | 需要回答的问题 | 核心证据 | 可能结论 |
|---|---|---|---|
| 信号方向 | 分数越高是否真的对应更高未来收益？ | daily RankIC、bucket lift、top-minus-bottom、valid/test 一致性 | `signal_inverted` / `weak_signal` / `usable_signal` |
| 分类质量 | `label_top30_5d` 是否能被模型区分？ | AUC、logloss、base rate、calibration | `classification_noise` / `calibration_bad` |
| 标签口径 | 5 日 open-to-close 标签是否匹配周频调仓和持有期？ | 1/5/10/20 日标签对照、fwd_xs_ret 分布 | `label_horizon_mismatch` |
| 样本 / 股票池 | 训练面板是否因 `sample_trainable_default` 使用了 live 不可得过滤？ | trainable panel vs pure universe funnel、label_valid 排除率 | `sample_filter_risk` |
| 候选池 | 高分股票如何从预测进入候选、组合、成交？ | prediction → universe → selected → order → filled funnel | `selection_funnel_issue` |
| 组合 | 回测差是否来自持股过少、权重过集中、卖不掉、现金约束？ | position_count、HHI、top weight、skip rate、cash/exposure | `portfolio_concentration_issue` |
| 成本 / 换手 | 收益是否被交易成本吞噬？ | turnover、commission/tax/slippage、cost/absolute PnL | `cost_turnover_issue` |
| 风格 / 市场 | 是否暴露在小市值、高波动、低流动性或特定市场阶段？ | log_mv、amount、vol、valuation、benchmark-relative return | `style_regime_issue` |
| 数据质量 | 是否有主键、缺失、分区、benchmark 或报告 artifact 异常？ | 现有 QA + 诊断自检 | `pipeline_issue` |

## 7. 功能需求

### FR-DIAG-1 运行身份冻结

诊断必须记录：

```json
{
  "diagnosis_version": "strategy1_model_diagnosis_v1",
  "strategy_id": "ml_pv_clf_v0",
  "run_id": "...",
  "backtest_id": "...",
  "model_id": "...",
  "feature_version": "...",
  "label_version": "...",
  "universe_version": "...",
  "generated_at": "..."
}
```

如果同一个 `run_id` 找不到唯一 selected model，诊断必须失败。

### FR-DIAG-2 信号方向与稳定性

按 `split_tag IN ('valid','test')` 计算：

- daily RankIC：使用 Spearman rank correlation，即先按每个 `trade_date` 对 score 与 `fwd_xs_ret_5d` 分别做日截面 rank，再计算 `CORR(score_rank, ret_rank)`。
- RankIC mean / std / ICIR / 正 IC 天数占比。
- 按年、按月、按回测窗口切片的 RankIC。
- top-minus-bottom：高分 bucket 平均未来收益减低分 bucket 平均未来收益。
- topN 平均 `fwd_xs_ret_5d`，其中 N 默认等于当前 `p_target_holdings`。

至少输出：

```text
daily_rank_ic.csv
rank_ic_summary.json
```

### FR-DIAG-3 分层收益、lift 与校准

按每个 `trade_date` 的预测分数日截面 quantile 做 5 分组和 10 分组，分别计算：

- 样本数；
- `label_top30_5d` 命中率；
- `fwd_xs_ret_5d` 均值 / 中位数；
- `fwd_ret_5d` 均值；
- 平均 score；
- bucket monotonicity；
- top bucket 与 bottom bucket 差值。

校准检查：

- score bucket 内实际 label rate；
- score 分布是否过窄；
- high score 与 actual positive rate 是否同向。

输出：

```text
score_bucket_lift.csv
score_calibration.csv
```

### FR-DIAG-4 标签 horizon 对照

用 `dws_stock_sample_daily` 中已有标签，比较同一预测分数对以下目标的排序关系：

- `fwd_xs_ret_1d`
- `fwd_xs_ret_5d`
- `fwd_xs_ret_10d`
- `fwd_xs_ret_20d`
- `label_top30_5d`
- `label_above_median_5d`

目标是判断当前 5 日 top30 分类标签是否适合周频选股。若 1/10/20 日上信号方向明显不同，后续再开标签实验。

输出：

```text
label_horizon_comparison.csv
```

### FR-DIAG-5 样本和股票池漏斗

按交易日和 split 统计：

```text
pure universe
-> sample_trainable_default
-> prediction rows
-> in_universe_default
-> selected candidate
-> order planned
-> filled trade
-> position held
```

必须显式量化：

- 仅沪深主板股票池覆盖率；
- `sample_trainable_default` 相对 `in_universe_default` 的排除率；
- 因 label_valid / full history / tradability 等条件导致的排除率；
- valid/test 预测是否依赖未来标签可得性过滤。

`sample_filter_risk` 的 10% 阈值只针对“不可解释排除率”，不能直接套用总排除率。不可解释排除率定义为：

```text
unexplained_exclusion_rate =
  max(total_exclusion_count - explained_exclusion_count, 0)
  / pure_universe_count
```

其中 `explained_exclusion_count` 至少包括上市天数不足、full history 不完整、ST/停牌/一字板等可交易性过滤，以及训练期标签构造需要的 `label_valid` 过滤；valid/test live 预测不可依赖未来标签可得性过滤，这类过滤不得计入可解释排除。

如果预测池使用了 live 预测时不可得的标签有效性过滤，诊断结论必须标记为 `sample_filter_risk`，不能静默继续调参。

输出：

```text
sample_universe_funnel.csv
candidate_funnel.csv
```

### FR-DIAG-6 特征暴露诊断

比较以下集合的特征分布：

- 全部预测股票；
- 高分 bucket；
- 低分 bucket；
- 入选候选；
- 实际持仓；
- 亏损贡献最大股票。

P0 至少覆盖当前模型已有特征：

```text
ret_1d, ret_5d, ret_20d, ret_60d,
mom_20_5, mom_60_20,
vol_20d, drawdown_20d,
amount_ma20_cny, amount_zscore_20d,
turnover_rate, volume_ratio,
pe_ttm, pb, ep_ttm, bp,
log_total_mv, log_circ_mv
```

如 `dws_stock_feature_fin_daily` 可用，可作为观察字段加入诊断，但 P0 不要求把财务特征纳入模型训练。

输出：

```text
feature_exposure_by_group.csv
top_loss_feature_profile.csv
```

### FR-DIAG-7 回测链路归因

基于 ADS 回测表，诊断：

- 快速亏损窗口；
- 回撤窗口；
- 亏损股票和持仓权重；
- 交易成本拆分：commission / tax / slippage；
- 换手率；
- 买入 / 卖出不可交易 skip；
- 现金和 gross exposure；
- 持仓数、单票最大权重、集中度 HHI。

输出：

```text
drawdown_window_diagnostics.csv
loss_window_trades.csv
portfolio_concentration.csv
cost_turnover_diagnostics.csv
```

### FR-DIAG-8 市场和风格阶段

以中证 1000 为主基准、沪深 300 为展示对比，诊断：

- 策略收益与基准收益相关性；
- 相对中证 1000 的 rolling excess return；
- 快速亏损窗口是否同时发生在中证 1000 下跌阶段；
- 持仓相对预测池的市值、流动性、波动、估值偏离。

行业暴露 P0 不强制，因为 `dim_stock_sw_industry_hist` 尚未落地；后续行业维表完成后再扩展行业归因。

输出：

```text
market_regime_diagnostics.csv
style_exposure_diagnostics.csv
```

### FR-DIAG-9 诊断结论分类

诊断脚本必须输出机器可读结论：

```json
{
  "primary_diagnosis": "signal_inverted | weak_signal | label_horizon_mismatch | sample_filter_risk | portfolio_concentration_issue | cost_turnover_issue | style_regime_issue | pipeline_issue | inconclusive",
  "confidence": "high | medium | low",
  "evidence": [
    {"metric": "valid_rank_ic_mean", "value": -0.10, "threshold": "..."},
    {"metric": "test_rank_ic_mean", "value": "...", "threshold": "..."}
  ],
  "recommended_next_experiments": []
}
```

阈值建议：

| 结论 | 建议阈值 |
|---|---|
| `signal_inverted` | valid 与 test 的 RankIC 均 < -0.03，且 score bucket top-minus-bottom 均 < 0。 |
| `weak_signal` | valid/test RankIC 绝对值均 < 0.02，bucket 不单调，AUC 接近 0.50。 |
| `label_horizon_mismatch` | 5d 信号弱，但 10d/20d 或 1d 对照明显更强，且 valid/test 同向。 |
| `sample_filter_risk` | 预测池相对 pure universe 的不可解释排除率 > 10%，或 valid/test 预测依赖 live 不可得标签过滤；不可解释排除率按 FR-DIAG-5 定义，不等同于总排除率。 |
| `portfolio_concentration_issue` | 亏损主要由少数持仓贡献，或 HHI / 单票权重过高。 |
| `cost_turnover_issue` | 总经济成本 / 绝对收益或绝对亏损 >= 20%。 |
| `style_regime_issue` | 亏损窗口与特定市值 / 波动 / 流动性暴露显著相关。 |

阈值是诊断门槛，不是交易规则。实现时必须写入 `diagnosis_summary.json`，便于后续复核。

## 11. 决策门槛

诊断完成后，按以下门槛决定下一步：

| 诊断结果 | 下一步 |
|---|---|
| valid/test 均稳定反向 | 新开“反向分数 / 反向标签”实验，不直接覆盖当前 baseline。 |
| 信号无信息量 | 优先改特征或标签，不优先调组合参数。 |
| 5d 标签不匹配 | 新开标签 horizon PRD/实验，比较 1d/10d/20d 或回归标签。 |
| 样本过滤存在 live 不可得风险 | 先修训练/预测面板口径，再谈模型质量。 |
| score 有信息量但回测差 | 优先调持股数、权重上限、调仓频率和风险约束。 |
| 成本 / 换手问题明显 | 做调仓频率、换手控制和持有期实验。 |
| 风格阶段问题明显 | 补 `dws_market_state_daily` 或风格中性/风险开关实验。 |
| 结论不充分 | 保留 baseline，不进入生产或参数固化。 |

## 12. 验收标准

本文 PRD 通过后，后续实现 PR 的验收标准为：

- 诊断脚本可针对当前 baseline run 生成完整 `model_diagnosis/` 本地和 GCS artifact。
- ADS summary `metrics_json` 能追踪诊断状态、版本、URI、主结论、置信度和 artifact manifest。
- `12_qa_model_diagnosis_outputs.sql` 全部通过。
- `diagnosis.md` 用中文总结主要结论，但每条结论能追溯到 `diagnosis_summary.json` 或 CSV。
- 诊断结论明确给出下一步实验建议，并标明哪些建议证据不足。
- 诊断本身不改变模型、预测、候选池、组合目标、回测成交、NAV 或绩效 summary 的核心回测指标。

## 13. 风险和边界

| 风险 | 处理 |
|---|---|
| 诊断后仍可能无法唯一定位原因 | 输出 `inconclusive`，不要强行给结论。 |
| valid/test 都只有一个历史窗口，仍有过拟合风险 | 后续实验必须保留固定切分，并可增加滚动年份验证。 |
| 当前预测池来自训练面板，可能带 label-valid 过滤 | 诊断必须量化该风险；若成立，先修面板口径。 |
| 行业归因暂不完整 | P0 用市值、流动性、波动、估值替代；行业维表落地后再扩展。 |
| AI 容易过度解释 | AI 只能引用 artifact；无证据必须写不可判断。 |

## 14. 结论

策略 1 下一步应先做模型质量诊断，而不是直接调参。该诊断 PRD 的核心价值是把“策略效果差”拆成可验证的证据链，并为后续 OQ-010 参数和模型迭代提供准入门槛。
