# PRD: 策略 1 验收门 v3

> 文档维护：GPT-5 Codex（最近更新 2026-06-07）

## 1. 背景

策略 1 已完成 BigQuery SQL runner、Cloud Run Python runner、LightGBM binary / regression、风险特征入模和整数手 ledger 等多轮实验。现有验收门分两层：

- v1：当前 Python search 的主写回门，偏向基础信号质量、2025 test、full-period 收益 / 回撤 / Sharpe 和 final holdout veto。
- v2：下一代生产门诊断，新增 eligible signal / executable、lot-aware 交易可行性和 final holdout 约束，但尚不是当前 Python search 的主写回门。

owner 要求定义 v3：保留 v1/v2 中必要的信号质量和执行可行性门，同时引入五个指数维度的“策略最大回撤同期超额”和 Excess Calmar Ratio，用于更直接衡量策略在自身最大回撤窗口内相对指数的风险收益质量。

## 2. 目标

1. 固化策略验收门 v3 的完整条件和公式，避免只在临时 CSV 中维护口径。
2. 将项目内年化、月化、日化等周期化收益默认口径统一为复利 / 几何换算。
3. 明确 10 / 20 / 30 / 40 持仓数组合均可参与 accepted，并分别定义单票最大权重。
4. 补回 v1/v2 中必须保留的 CV、valid、test、score orientation、final holdout、诊断和交易可行性约束。
5. 为后续实现 `model_acceptance_contract_v3.yml`、诊断脚本和 QA SQL 提供可执行需求。

## 3. 非目标

- 不在本 PRD 中直接修改 Cloud Run runner、acceptance.py、ADS 表或 BigQuery SQL。
- 不重新回测历史候选，不自动把此前 rejected 候选改为 accepted。
- 不把下载目录中的个人 CSV 文件作为项目事实来源；CSV 只能作为临时导出或人工审阅副本。
- 不改变策略训练数据、特征集合、模型族或 ledger 执行语义。

## 4. 核心定义

### 4.1 复合年化收益

默认年化收益使用复利口径：

```text
compound_annual_return = (ending_value / beginning_value) ^ (252 / return_period_count) - 1
```

其中：

- `return_period_count` 是有效日收益期数，通常等于 NAV 点数减 1。
- A 股默认年交易日数使用 `252`。
- 若使用指数价格，`ending_value / beginning_value` 对应期末收盘价除以期初收盘价。

### 4.2 月化 / 日化收益

项目内后续凡提到月化、日化等周期化收益，默认同样使用复利 / 几何换算：

```text
compound_period_return = (ending_value / beginning_value) ^ (target_period_count / return_period_count) - 1
```

默认约定：

- 年化：默认使用 `252` 作为标准年化交易日因子；这只是换算约定，不表示 A 股每个自然年固定有 252 个真实开市日。
- 样本期：`return_period_count` 必须使用该样本区间内实际可用的有效日收益期数，不能写死为 252。
- 月化：若按交易日口径，默认使用 `21` 作为标准月化交易日因子；这只是换算约定，不表示每个自然月固定有 21 个真实开市日。
- 日化：`target_period_count = 1`。
- 自然月 / 自然年口径：若报告明确使用自然日历口径，必须单独标注，并说明有效期数和换算方法。

禁止在未标注的生产验收门中混用 `平均日收益率 * 252` 作为年化收益。

### 4.3 策略最大回撤

```text
strategy_max_drawdown = min(strategy_nav / running_peak_nav - 1)
```

策略最大回撤窗口定义为：

```text
strategy_peak_date -> strategy_trough_date
```

其中 `strategy_peak_date` 是最大回撤发生前最近一次策略 NAV 创新高日期，`strategy_trough_date` 是最大回撤谷底日期。

### 4.4 指数同期回撤

指数同期回撤使用策略自身最大回撤窗口内的指数涨跌幅：

```text
index_same_window_return = index_close(strategy_trough_date) / index_close(strategy_peak_date) - 1
```

说明：该值可能为正，也可能为负。字段名沿用“回撤”，但计算上是指数在该窗口内的 close-to-close return。

### 4.5 策略最大回撤同期超额

v3 统一使用新字段名：`策略最大回撤同期超额`。

```text
strategy_max_drawdown_same_window_excess
  = strategy_max_drawdown - index_same_window_return
```

解释：

- 若该值为负，表示策略在自身最大回撤窗口内比指数更差。
- 若该值为正，表示策略在自身最大回撤窗口内比指数更抗跌，或指数涨跌幅更差。
- 旧名称 `策略同期回撤超额` 停止使用。

### 4.6 策略超额复合年化收益

```text
strategy_compound_annual_excess_return
  = strategy_compound_annual_return - index_compound_annual_return
```

所有指数对比均使用同一回测全周期 `2024-01-02 ~ 2026-04-30` 的复合年化收益。

### 4.7 Excess Calmar Ratio

```text
excess_calmar_ratio
  = strategy_compound_annual_excess_return
    / abs(strategy_max_drawdown_same_window_excess)
```

v3 的五指数相对门必须先要求同一指数下：

```text
strategy_compound_annual_excess_return > 0
```

特殊规则：

- 若 `strategy_compound_annual_excess_return <= 0`，该指数维度不得通过，即使 `strategy_max_drawdown_same_window_excess > 0` 也不得 direct-pass。
- 若 `strategy_compound_annual_excess_return > 0` 且 `strategy_max_drawdown_same_window_excess > 0`，该指数维度直接视为满足 Excess Calmar 条件，不再计算 ratio。
- 若 `strategy_max_drawdown_same_window_excess = 0`，实现层必须标记为 `zero_excess_drawdown_denominator`，不自动通过，避免除零。
- 若 `strategy_compound_annual_excess_return > 0` 且 `strategy_max_drawdown_same_window_excess < 0`，正常计算 ratio，并要求 `excess_calmar_ratio > 1`。

### 4.8 Calmar Ratio

```text
calmar_ratio = strategy_compound_annual_return / abs(strategy_max_drawdown)
```

v3 要求：

```text
calmar_ratio > 1
```

## 5. v3 验收门

### 5.1 回测范围与主基准

| 条件 | 要求 |
|---|---|
| 回测全周期 | `2024-01-02 ~ 2026-04-30` |
| 主 benchmark | 上证指数，即 `000001.SH` |
| 指数对比集合 | 上证50、沪深300、中证1000、上证指数、深证成指 |

### 5.2 候选资格门

| 条件 | 要求 |
|---|---|
| 候选池 | 必须来自 `eligible signal pool` |
| 可执行性 | 必须是 `eligible executable` |
| Score orientation | `identity` 或 `reverse_probability` |
| 诊断状态 | 不能是 `signal_inverted`、`unusable_signal`、`execution_infeasible` 等硬拒绝状态 |

### 5.3 信号质量门

| 条件 | 要求 |
|---|---|
| CV 状态 | `passed` |
| Valid signal | `stable` |
| Valid RankIC | `> 0` |
| Valid top-bottom | `> 0` |
| Test RankIC | `> 0` |
| Test top-bottom | `> 0` |

说明：v3 不允许只看回测收益接受候选；valid/test 的方向性和横截面排序质量必须保留。

### 5.4 收益与风险门

| 条件 | 要求 |
|---|---|
| Sharpe | `>= 0.70` |
| Calmar Ratio | `> 1` |

### 5.5 Final holdout 门

| 条件 | 要求 |
|---|---|
| Final holdout 交易日数 | `>= 40` |

Final holdout 在 v3 中只作为最小样本长度门，确保最新留出段有足够交易日可供审计；不再设置 final holdout 收益、超额收益或硬 veto 门。

### 5.6 持仓数与单票最大权重

| 目标持仓数 | 单票最大权重 | v3 状态 |
|---:|---:|---|
| 10 | 15.0% | 可参与 accepted |
| 20 | 7.5% | 可参与 accepted |
| 30 | 5.0% | 可参与 accepted |
| 40 | 5.0% | 可参与 accepted |

说明：

- 10 只组合不再沿用 5% 单票上限，因为 10 * 5% 最多只能部署 50% 仓位，天然不公平。
- 20 只组合使用 7.5% 单票上限，允许在候选不足或现金约束下更接近满仓。
- 30 / 40 只组合保持 5% 单票上限。
- 所有组合仍必须通过 100 股整数手、现金、持仓唯一性、skip reason 和实际成交约束。

### 5.7 五指数 Excess Calmar 门

对以下五个指数分别计算：

- 上证50
- 沪深300
- 中证1000
- 上证指数
- 深证成指

任一指数同时满足以下条件，即通过五指数 Excess Calmar 门：

```text
strategy_compound_annual_excess_return > 0
```

并且满足以下任一条件：

```text
strategy_max_drawdown_same_window_excess > 0
```

或：

```text
excess_calmar_ratio > 1
```

其中：

```text
excess_calmar_ratio
  = strategy_compound_annual_excess_return
    / abs(strategy_max_drawdown_same_window_excess)
```

注意：

- `strategy_compound_annual_excess_return` 必须使用策略和对应指数的复合年化收益相减。
- `strategy_max_drawdown_same_window_excess` 必须使用策略最大回撤窗口，不是全周期指数最大回撤。
- 不允许用一个指数满足正超额年化收益、另一个指数满足 Excess Calmar / 回撤同期超额后拼接通过；必须同一个指数同时满足正超额年化收益和相对风险收益条件。
- 若五个指数全部不满足，则候选不得 accepted。

### 5.8 Lot / 交易可行性门

| 条件 | 要求 |
|---|---|
| 100 股整数手 | 买入必须按 100 股整数手执行 |
| odd-lot 卖出 | 清仓 odd-lot 可全额卖出；部分卖出按实现约束处理 |
| 现金约束 | 不得产生不可解释的负现金 |
| 持仓唯一性 | 每日每股持仓事实唯一 |
| skip reason | 跳过买卖必须有可审计 reason |
| 执行状态 | 不得出现未知订单状态或不可解释成交 |

v3 必须基于 lot-aware ledger 输出，不应继续使用旧 FLOAT shares reference 作为 production acceptance 依据。

## 6. 推荐实现方案

### 6.1 Contract 配置

新增版本化配置：

```text
configs/strategy1/model_acceptance_contract_v3.yml
```

建议结构：

```yaml
version: model_acceptance_contract_v3
period:
  start_date: 2024-01-02
  end_date: 2026-04-30
benchmark:
  primary: 000001.SH
  excess_calmar_indices:
    - 000016.SH
    - 000300.SH
    - 000852.SH
    - 000001.SH
    - 399001.SZ
annualization:
  method: compound
  trading_days_per_year: 252
holdings:
  - target_holdings: 10
    max_single_weight: 0.15
    acceptance_status: accepted_eligible
  - target_holdings: 20
    max_single_weight: 0.075
    acceptance_status: accepted_eligible
  - target_holdings: 30
    max_single_weight: 0.05
    acceptance_status: accepted_eligible
  - target_holdings: 40
    max_single_weight: 0.05
    acceptance_status: accepted_eligible
thresholds:
  min_sharpe: 0.70
  min_calmar_ratio: 1.0
  min_excess_calmar_ratio: 1.0
  min_final_holdout_days: 40
```

### 6.2 Python acceptance 逻辑

`acceptance.py` 或后续 v3 诊断脚本必须：

1. 从 contract 读取所有阈值，不在代码中硬编码。
2. 用复合年化收益替代 `AVG(daily_return) * 252`。
3. 同时输出 full-period、2025 test、final holdout 和五指数 Excess Calmar 结果。
4. 对 `strategy_max_drawdown_same_window_excess > 0` 做 direct-pass 标记。
5. direct-pass 前必须先确认同一指数下 `strategy_compound_annual_excess_return > 0`。
6. 对 `strategy_max_drawdown_same_window_excess = 0` 做除零保护并输出显式诊断。
7. 输出每个指数的：指数同期回撤、指数最大回撤、指数总收益、指数复合年化收益、策略最大回撤同期超额、策略超额回撤、策略超额总收益、策略超额复合年化收益、Excess Calmar Ratio。

### 6.3 QA SQL

建议新增：

```text
sql/ml/strategy1/24_qa_acceptance_gate_v3_outputs.sql
```

QA 至少覆盖：

- contract hash / version 已写入 artifact 或 registry。
- 年化收益字段标记为 compound，且没有使用 simple annualization 标记。
- 10/20/30/40 持仓数和权重来自 contract。
- 五指数 Excess Calmar 至少输出 5 行明细。
- `策略最大回撤同期超额` 字段存在，旧字段名 `策略同期回撤超额` 不再作为主输出。
- 五指数相对门通过的指数必须同一指数下 `strategy_compound_annual_excess_return > 0`。
- `strategy_max_drawdown_same_window_excess > 0` 的指数 direct-pass 状态可审计，且 direct-pass 不得绕过正超额年化收益要求。
- final holdout 交易日数输出且满足最小样本长度要求。
- lot-aware ledger 证据存在，旧 FLOAT shares reference 不得作为 production acceptance 依据。

3. 新增只读 v3 诊断脚本，先对历史 20 个 Python Top5 候选做 replay，不写 ADS accepted。

6. 接入主写回门前，保留 v1/v2 诊断输出，避免口径切换后失去横向可比性。

## 9. 验收标准

v3 实现完成后，至少满足：

- `model_acceptance_contract_v3.yml` 可被 runner / 诊断脚本加载并记录 hash。
- 一个历史候选 replay 能输出完整 v3 gate summary。
- 五指数 Excess Calmar 明细完整且公式可复核。
- 年化收益全部使用复合年化收益。
- 10/20/30/40 持仓配置来自 contract，而不是代码硬编码。
- v1/v2 的信号质量门没有被回测表现门替代。
- QA SQL 失败时能明确指出缺失字段、错误口径或未注入 contract。

## 10. 风险与注意事项

- 五指数任一满足可能放宽风格约束，因此必须同时保留信号质量门、Sharpe、Calmar Ratio 和 final holdout 交易日数门。
- `策略最大回撤同期超额 > 0` direct-pass 是风险收益补充证据，不应绕过信号质量、交易可行性门或同一指数下的正超额复合年化收益要求。
- 复合年化收益切换会改变此前使用 `AVG(daily_return) * 252` 的数值，历史表格需标注旧口径或重算。
- 主 benchmark 上证指数 `000001.SH` 当前不在项目 BigQuery `dwd_index_eod` 可用指数集合中；实现前需要决定将其纳入 `dim_index/dwd_index_eod`，或在诊断脚本中明确使用外部行情源并记录 source。
