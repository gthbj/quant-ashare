# PRD：OQ-010 策略 1 交易成本口径

> 文档维护：GPT-5（最近更新 2026-06-02）
> 状态：草案 v0.1，待 owner review。
> 范围声明：本文只解决 OQ-010 中的交易成本参数口径，不解决调仓频率、持股数、单票权重上限、特征/标签/选股口径。
> 上游依据：`docs/策略1-ml_pv_clf_v0-runner设计.md`、`docs/prd/PRD_20260601_02_策略1BQML回测闭环.md`、`.agent/memory/OPEN_QUESTIONS.md` OQ-010。

## 1. 背景

策略 1 runner 当前在 `sql/ml/strategy1/08_run_backtest.sql` 与 `09_build_metrics_and_report_inputs.sql` 中使用单一参数：

```sql
DECLARE p_cost_bps FLOAT64 DEFAULT 30.0;
```

该参数把佣金、印花税和滑点合并为单边固定 bps。这个实现能快速跑通 v0，但不能表达真实 A 股成本结构：

- 佣金按成交金额双边收取，owner 指定为“万一免五”。
- 证券交易印花税是卖出侧成本，不能对买入侧同样计提。
- 滑点更适合体现为成交价格偏移，而不是简单记在 `fee_cny`。

因此需要把 OQ-010 的交易成本子项从“待确认示例值”收敛为可实现、可审计、可复现实验配置。

## 2. 外部口径

证券交易印花税采用当前常用 A 股口径：卖出侧 5 bps。依据为财政部、税务总局 2023 年第 39 号公告，自 2023-08-28 起证券交易印花税减半征收。官方来源：[财政部公告](https://www.mof.gov.cn/jrttts/202308/t20230828_3904235.htm)、[税务局政策库](https://shanghai.chinatax.gov.cn/tax/zcfw/zcfgk/yhs/202308/t468451.html)。

本文不额外建模过户费、经手费、证管费等细项。P0 策略 1 小资金基线把这些较小项目并入滑点缓冲；若后续需要券商/交易所级精细成本，再新开成本 profile。

## 3. 目标与非目标

### 3.1 目标

1. 固化策略 1 P0 默认交易成本 profile。
2. 将成本从单一 `p_cost_bps` 升级为分项参数：佣金、印花税、滑点。
3. 明确买入和卖出的现金影响公式。
4. 明确现有 `30 bps` v0 回测与新成本 profile 的可比性边界。
5. 给后续实现 PR 提供 SQL、报告和 QA 验收标准。

### 3.2 非目标

- 不在本文实现 SQL 修改。
- 不改变策略 1 原始 v0 模型特征白名单。
- 不决定 OQ-010 剩余项：调仓频率、持股数、单票权重上限。
- 不模拟真实券商所有费用明细。
- 不处理融资融券、ETF、可转债、港股通或场外基金费用。

## 4. 成本 profile 决策

默认 profile：

```text
cost_profile_id = cn_a_share_wanyi_no_min_slip5_v20260602
```

| 子项 | 参数 | 默认值 | 方向 | 说明 |
|---|---:|---:|---|---|
| 佣金 | `commission_bps` | 1.0 | 买入 + 卖出 | “万一”，即成交金额的 0.01%。 |
| 最低佣金 | `min_commission_cny` | 0.0 | 买入 + 卖出 | “免五”，不设置 5 元最低收费。 |
| 印花税 | `stamp_tax_buy_bps` | 0.0 | 买入 | 买入不收证券交易印花税。 |
| 印花税 | `stamp_tax_sell_bps` | 5.0 | 卖出 | 当前 A 股常用卖出侧 0.05%。 |
| 滑点 | `slippage_buy_bps` | 5.0 | 买入 | 买入成交价向上偏移 5 bps。 |
| 滑点 | `slippage_sell_bps` | 5.0 | 卖出 | 卖出成交价向下偏移 5 bps。 |

经济含义：

| 方向 | 显性费用 | 隐性滑点 | 总经济成本 |
|---|---:|---:|---:|
| 买入 | 1 bps | 5 bps | 6 bps |
| 卖出 | 1 + 5 = 6 bps | 5 bps | 11 bps |
| 完整买卖往返 | 7 bps | 10 bps | 17 bps |

说明：

- 旧 runner 的 `p_cost_bps=30.0` 是单边一揽子示例值，不再作为 OQ-010 成本默认口径。
- 新 profile 下，成本方向不对称，不能再用一个 `p_cost_bps` 精确表达。
- 历史回测 `bt_s1_bqml_20260601_01` 保留旧成本口径，不回写修改；新 run 必须使用新的 `cost_profile_id`。

## 5. 功能需求

### FR-COST-1 参数化

后续实现 PR 应在 `08_run_backtest.sql` 与 `09_build_metrics_and_report_inputs.sql` 中新增成本参数：

```sql
DECLARE p_cost_profile_id STRING DEFAULT 'cn_a_share_wanyi_no_min_slip5_v20260602';
DECLARE p_commission_bps FLOAT64 DEFAULT 1.0;
DECLARE p_min_commission_cny FLOAT64 DEFAULT 0.0;
DECLARE p_stamp_tax_buy_bps FLOAT64 DEFAULT 0.0;
DECLARE p_stamp_tax_sell_bps FLOAT64 DEFAULT 5.0;
DECLARE p_slippage_buy_bps FLOAT64 DEFAULT 5.0;
DECLARE p_slippage_sell_bps FLOAT64 DEFAULT 5.0;
```

`p_cost_bps` 后续只允许作为兼容字段保留，不再作为默认撮合成本来源。

### FR-COST-2 买入现金影响

买入使用 `exec_open` 作为原始开盘价，滑点通过成交价体现：

```text
fill_price = exec_open * (1 + slippage_buy_bps / 10000)
gross_turnover_cny = filled_shares * fill_price
commission_cny = max(gross_turnover_cny * commission_bps / 10000, min_commission_cny)
stamp_tax_cny = gross_turnover_cny * stamp_tax_buy_bps / 10000
cash_effect_cny = -(gross_turnover_cny + commission_cny + stamp_tax_cny)
fee_cny = commission_cny + stamp_tax_cny
```

当前默认 `min_commission_cny=0`，所以小额订单不会被 5 元最低佣金放大。

### FR-COST-3 卖出现金影响

卖出同样使用滑点调整成交价：

```text
fill_price = exec_open * (1 - slippage_sell_bps / 10000)
gross_turnover_cny = filled_shares * fill_price
commission_cny = max(gross_turnover_cny * commission_bps / 10000, min_commission_cny)
stamp_tax_cny = gross_turnover_cny * stamp_tax_sell_bps / 10000
cash_effect_cny = gross_turnover_cny - commission_cny - stamp_tax_cny
fee_cny = commission_cny + stamp_tax_cny
```

卖出侧显性费用为佣金 1 bps + 印花税 5 bps；滑点已通过更低的 `fill_price` 反映。

### FR-COST-4 现金约束与买入缩放

08 ledger 买入缩放必须按新成本公式计算可用现金约束。买入预算不能只除以 `1 + p_cost_bps/10000`，而应使用：

```text
required_cash = gross_turnover_cny + commission_cny + stamp_tax_cny
```

若买入需求超过可用现金，仍按当前 v1 ledger 规则等比例缩放买入腿，且缩放后必须满足：

```text
cash_cny >= -1
gross_exposure <= 1.005
```

### FR-COST-5 报告和元数据

`ads_backtest_performance_summary.metrics_json` 必须记录：

```json
{
  "cost_profile_id": "cn_a_share_wanyi_no_min_slip5_v20260602",
  "commission_bps": 1.0,
  "min_commission_cny": 0.0,
  "stamp_tax_buy_bps": 0.0,
  "stamp_tax_sell_bps": 5.0,
  "slippage_buy_bps": 5.0,
  "slippage_sell_bps": 5.0,
  "effective_buy_cost_bps": 6.0,
  "effective_sell_cost_bps": 11.0,
  "round_trip_cost_bps": 17.0
}
```

`ads_backtest_performance_summary.cost_bps` 作为兼容字段，应写入 `round_trip_cost_bps=17.0`，并在 README / 报告中明确该字段不是单边成本。长期更干净的方案是新增 `cost_profile_id` 物理列，但 P0 不强制改 ADS 契约。

### FR-COST-6 报告展示

`scripts/strategy1/render_report.py` 后续应在报告中展示成本 profile，至少包括：

- 佣金：万一免五。
- 印花税：卖出 5 bps。
- 滑点：买入 5 bps、卖出 5 bps。
- 买入、卖出、往返总经济成本。
- 与旧 `30 bps` 一揽子成本的差异说明。

## 9. 风险和边界

| 风险 | 处理 |
|---|---|
| 滑点“常用值”不是市场不变真理 | P0 固定 5 bps 做默认基线；后续可加 0/5/10 bps 敏感性实验。 |
| `cost_bps` 语义历史上是单边一揽子 | 新 profile 写 `metrics_json` 作为事实来源，`cost_bps` 仅兼容展示。 |
| `fee_cny` 不含滑点可能让总成本低估 | 报告和 summary 必须同时给显性费用与隐性滑点。 |
| 小额订单过多时免五影响很大 | 本文已按 owner 指定设置 `min_commission_cny=0`，不做 5 元最低费。 |

## 10. 结论

OQ-010 的交易成本子项建议关闭为：

```text
佣金：万一免五
印花税：卖出 5 bps，买入 0
滑点：买入 5 bps，卖出 5 bps
成本 profile：cn_a_share_wanyi_no_min_slip5_v20260602
```

OQ-010 其余参数（调仓频率、持股数、单票权重上限、特征/标签/选股口径）仍保持 open，后续单独决策和实现。
