# PRD：策略 1 自上而下整手组合构造（top-down lot-aware construction）

> 状态：草案，待 owner review。
> 范围声明：新组合构造规则 + 新 ledger 语义版本 + research-only 验证；不动 `ledger_exec_v1_lot100`
> 语义、不改全局默认 `tail_risk_profile_id`、不重训模型、不 promotion。
> 关联：PR #186（PRD_09 分析，执行层现金拖累发现）、PR #179（P1/P2 Overlay A/B）、
> KNOWN_CONSTRAINTS 中 P1 profile 约束与 lot100 交易执行约束、OQ-010。

## 1. 背景

PRD_09 分析（PR #186）经现金交叉核验（NAV `cash_cny` vs `1-Σweight` 最大差 `2.22e-16`、
差异日 0）确认：official continuous baseline 在"初始资金 10 万 + 100 股整手 + 等权 5% +
无现金再分配（`cash_redistribution: none_v1`）"的参数组合下存在**结构性现金拖累**——
全周期 `BUY_SKIPPED_BELOW_LOT` 690 笔（约 25% 买单、123 个执行日），平均持仓 16.3/20，
现金权重均值 29.4%，265 个交易日现金过半，最少持仓 0（2021-12-27 当日 20 笔 BUY 全部
below-lot 跳过、现金 95.7%）。组合的实际暴露由"股价 vs 单仓预算"的粒度关系随机决定，
既有"满仓"叙事不成立，且回测结果不具资金规模不变性。

Owner 决策（2026-06-11）：真实部署规模即 10 万；不采用"修复等权 + 现金再分配"路线，
改为**重新设计构造规则**——顺着 lot 粒度做仓位，而非对抗它。本 PRD 即该设计的契约。

支撑证据（PRD_09 转换阶梯）：信号在排名前列是平的（Top20 IR `0.544` ≈ Top50 IR `0.491`
≈ top-decile IR `0.510`），因此"跳过买不起/被风控标记的票、用排序下一名顶上"的替换成本
接近零；等权 vs 分数加权差仅 `0.058` IR，权重精度本身不值钱。

## 2. 构造规则（核心契约）

记净值 `V`、股价 `p`、一手金额 `L = 100p`：

### 2.1 参数

| 参数 | 默认值 | 含义 |
|---|---|---|
| `position_floor_count` | 20 | 仓位门槛基数，仅用于推导最小仓位权重 |
| `min_position_weight` | `1 / position_floor_count = 5%` | 每个**新开**仓位的最小权重 |
| `walk_depth` | 50 | 行走/保留的排名深度上限 |
| 单票上限 | **无**（owner 决策 2026-06-11：移除原 7.5% / 讨论稿 20% 上限） | 见 §7 风险 |

原 `target_holdings=20` 退役为非约束；实际持仓数降级为观测指标
`realized_holdings_count`（记录、报告，不参与任何构造判断）。

### 2.2 行走算法

```
按分数排序自上而下走到 walk_depth，对每只候选：
  1. 已持仓 → 保留当前持仓（不强制加仓、不强制卖出），不消耗新现金，继续；
  2. 被 P1 标记（individual_risk_guard_v0 六条规则，含 required-field-NULL）→ 跳过，
     记录 (sec_code, 触发规则, 当时排名)，继续；
  3. 未持仓且未标记 → 最小手数 m = ceil(min_position_weight × V / L)：
       若 剩余现金 ≥ m × L → 买入 m 手；
       否则 → 跳过（更便宜的下一名可能仍装得下），继续。
新开仓终止条件：剩余现金 < min_position_weight × V，或走到 walk_depth。
余现金保留，不二次分配。
```

### 2.3 卖出规则

已持仓票当期排名 > `walk_depth` → **卖出**（netting 语义沿用 v1：清仓允许 odd-lot，
部分卖出向下取整到手）。卖出失败（停牌/跌停等不可交易）沿用 v1 的
`SELL_SKIPPED_UNTRADABLE` / `PENDING_SELL_CARRY` 顺延语义。

**豁免边界（消歧义，QA 据此断言）**："持仓豁免"仅指排名 ≤ `walk_depth` 的已持仓票
不因 P1 标记或可负担性被强制卖出；**不存在"超深度豁免"**——超深度票在调仓后仍在
持仓中，唯一合法解释是当期卖出失败记录，QA 必须能将每一笔追溯到对应的
`SELL_SKIPPED_*` / `PENDING_SELL_CARRY`，否则判定规则失效。

> `walk_depth` 是本 PRD 引入的唯一新增自由参数。设 50 的依据：PRD_09 证明 top-50
> 信号与 top-20 持平；它同时是买入行走深度与卖出保留阈值，避免"卖出排名 25 的持仓、
> 转头买入排名 35 的新票"的不一致。Phase 0 对 {30, 50} 做敏感性，仅报告不反选。

### 2.4 P1 绑定的执行语义（实现红线）

P1 过滤**必须以 §2.2 行走中"跳过→下一名顶上"的替换语义实现**，被跳过候选的资金
自动流向后续候选。**禁止**复用 ledger 层 `BUY_SKIPPED_TAIL_RISK`（跳过→现金闲置）
语义实现本构造的 P1 过滤——PR #179 A1 臂 CAGR 12%→1.5% 的损耗主因即现金闲置，
若复用旧语义将在新构造中复活该拖累，并使 Phase 0 双臂对比失效。

此绑定是**本构造规格的组成部分**，不改变全局默认 `tail_risk_profile_id=diagnostic_only`，
不触碰 KNOWN_CONSTRAINTS 中"owner 新决策前不得把 `individual_risk_guard_v0` 设为默认
profile"约束（v1 路径与其他 run 不受影响）。

### 2.5 已接受的设计性质（记录在案，验证时观测，不再讨论）

1. **仓位大小与信号强度脱钩**：单仓权重 = 整手粒度的函数（@10 万：50 元以下股票
   ≈5-6%，高价股一手多少占多少），重仓的是贵的不是最看好的。依据：阶梯证明排序内
   权重分配价值 ≈ 0。
2. **无单票上限**（owner 决策）：高价股一手即可形成大权重仓位（@10 万：500 元股
   一手 = 50%）。`max_realized_weight` 为必报观测指标（见 §4/§5），P1 绑定为部分缓释。
3. **资金规模显式依赖**：本构造为 10 万真实部署口径设计，回测结果只对该量级成立；
   `initial_capital` 进入 run 口径记录。

### 2.6 执行与现金口径（实现边界）

1. **可负担性按含费口径判定**：行走中买入判断为
   `m × L × (1 + 买入成本费率) ≤ 剩余可用现金`，买入成本费率按既有成本 profile
   （佣金万一免五 + 买入滑点 5bps）；终止条件中的 `min_position_weight × V` 同样按
   含费口径取整估算。
2. **同日卖出回款计入可用现金**：当期可用现金 = 期初现金 + 当期卖出**净**回款
   （扣印花税/佣金/卖出滑点）− 已分配买入金额。行走规划按"计划卖出全部成交"的
   预计净回款排程（资金同日可用，沿用 v1 现金口径）。
3. **卖出失败时的买入降级**：若执行时部分卖出未成交（`PENDING_SELL_CARRY` 等）导致
   实际可用现金不足，按**排名从低到高整笔放弃**计划买入，并记录放弃审计；
   **禁止缩股数成交**——v2 中不得出现 `FILLED_SCALED_CASH`，因为缩量买入会击穿
   5% 仓位门槛、破坏整手语义。这是与 v1"买单按可用现金缩放"的显式分歧点。
4. **构造输入契约**：行走读取**全量带名次候选**（按 `rank_raw` 升序至 `walk_depth`），
   **不得**以旧 `is_selected_candidate`（Top-N 标记）作为输入过滤——该字段在 v2 中
   仅保留为兼容产物。候选层（`05`）为 v2 run 必须以启用 P1 标记的 profile 运行；
   若误以 `diagnostic_only` 输入运行，候选表无 `tail_risk:*` 标记，P1 过滤会静默
   退化为 no-op，QA 必须以"标记存在性断言"拦截（见 §5.3）。

## 4. Phase 0：paper 原型（先于一切 ledger 工程）

复用 PRD_09 转换阶梯机械（只读 BigQuery + 本地 pandas），在 10 万 + 整手约束下实现
§2 构造的 paper 版本，**双臂**：

- **T1**：完整规则（含 P1 替换过滤）；
- **T0**：同规则但无 P1 过滤（对照臂）。

同一 synthetic prediction 流（run id 从记忆解析，禁止硬编码）、同 biweekly 调仓日、
PIT 同 P2 约定、零成本 + 20bps 两档。可负担性判定按 §2.6 含费口径（20bps 档）；
卖出失败降级与 `PENDING_SELL_CARRY` 路径 paper 不模拟，写入局限声明（该偏差由
Phase 2 真实 ledger 与 paper 的对照量化，参照 PRD_09 恒等校验先例）。

**输出**（每臂）：CAGR / 年化波动 / IR vs `000852.SH` / MaxDD（含 peak/trough）/ Calmar /
2024-01-01~02-07 crunch 段超额 / 余现金时间序列（均值/峰值）/ `realized_holdings_count`
分布 / `max_realized_weight` 分布（均值/P95/最大值）/ 换手 / P1 跳过计数（T1）/
`walk_depth` {30,50} 敏感性；对照组：现行 official baseline 与 PRD_09 的满仓等权 paper L3。

**预登记判读**：

| 观测 | 解读 |
|---|---|
| T1 与 T0 全周期 CAGR 差 < 1pp 且 T1 crunch 段超额显著优于 T0 | P1 绑定成立（替换语义下成本≈0、危机保护为正），Phase 1 按 §2.4 实现 |
| T1 全周期显著差于 T0（>2pp CAGR） | P1 替换成本假设被证伪，回 owner 重议绑定 |
| 双臂 MaxDD 均显著差于 official（如深 >15pp）且 Calmar 明显恶化 | 新构造的满仓代价超预期，结果交 owner 决定是否继续 Phase 1 |
| `max_realized_weight` 尾部极端（如 >40% 频繁出现） | 如实报告，owner 复核无上限决策（不自行加上限） |

Phase 0 结果**无论好坏都不阻塞 Phase 1 的代码准备**，但 Phase 2 重跑前 owner 须见过
Phase 0 报告。

## 5. Phase 1：ledger v2 实现

1. 新 ledger 语义版本 **`ledger_exec_v2_lot100_topdown`**（参照 lot100 当年路径：新版本
   + 新 QA，不动 v1）；resume 语义沿用既有 resume policy 契约并适配新版本号。
2. 组合构造从"等权 target 20 只"改为 §2 行走算法；`position_floor_count` /
   `min_position_weight` / `walk_depth` / `initial_capital` 全部进 experiment 参数与
   run 口径记录。
3. 审计与 QA（新增 `qa_topdown_construction_outputs`，登记 catalog）：
   - 每个新开仓位在开仓时点权重 ≥ `min_position_weight`（容差仅来自当日价格波动）；
   - 整手约束（shares % 100 == 0，清仓 odd-lot 例外沿用 v1）；
   - P1 标记票零**新增**持仓；行走跳过记录（票/规则/排名）完整可查；
   - **标记存在性断言**（防 §2.6.4 的 no-op 退化，参照 #179 guard-effectiveness 先例）：
     全窗口候选层 `tail_risk:*` 标记数 > 0，且 v2 run 的 profile 参数为启用标记的取值；
   - **输入完整性断言**：构造审计记录每调仓日实际评估的候选数，
     断言 ≥ min(`walk_depth`, 当日候选总数)，拦截误用 `is_selected_candidate` Top-N 截断；
   - 调仓后余现金 < `min_position_weight × V`（或走满 `walk_depth` / 卖出失败降级的豁免记录）；
   - 已持仓豁免审计：被保留的 P1 标记票（rank ≤ `walk_depth`）追溯到"持仓豁免"；
     **超深度票仍持仓必须追溯到当期 `SELL_SKIPPED_*` / `PENDING_SELL_CARRY`**（§2.3
     豁免边界，不存在超深度豁免）；
   - 零 `FILLED_SCALED_CASH` 断言（§2.6.3）；卖出失败导致的整笔放弃买入有审计记录；
   - `realized_holdings_count` / `max_realized_weight` 逐调仓日落入 metrics。
4. 单测 + 包入口 + runbook 更新；retired linter / catalog caller 检查照常。

## 7. 风险与控制

| 风险 | 控制 |
|---|---|
| 无单票上限 → 高价股单仓极端集中（@10 万：500 元股一手=50%） | owner 已知并接受（2026-06-11 决策）；`max_realized_weight` 必报；P1 绑定缓释崩塌型风险；Phase 0 预登记"如实报告交 owner 复核"条款 |
| P1 误用旧"跳过留现金"语义实现 → 复活 A1 现金拖累 | §2.4 实现红线 + Phase 0 双臂对照 + QA 余现金断言 |
| 低价替换倾斜（行走跳贵买贱） | Phase 0 报告持仓的价格/市值分布对比 official；P1 市值规则托底 |
| 卖出阈值与买入深度共用 `walk_depth` 的耦合 | 唯一新增参数、{30,50} 敏感性仅报告；不合适由 owner 在 Phase 0 后调整 |
| MaxDD 较 official 显著加深（拆掉意外现金刹车的已知方向） | 预期内（满仓反事实 paper L3 已示 -57.8%）；预登记条款交 owner 判断，不回头偷偷加仓位上限 |
| 结果被解读为 accepted / 新官方口径自动生效 | §6 显式声明：取代 official 口径是 owner 决策，且不等于 accepted |

## 8. 实施顺序

1. Phase 0 paper 双臂原型 + 报告（预登记判读先写入报告再填数）。
2. Phase 1 ledger v2 + QA + 测试（可与 Phase 0 报告评审并行准备）。
3. owner 过目 Phase 0 报告后 → Phase 2 research-only 重跑 + 三方对比表。
4. 更新记忆 / TODO；新官方口径与后续（暴露条件化、对冲评估）由 owner 另行决策。
