# PRD: 策略 1 中文回测报告与归因分析

> 文档维护：GPT-5（最近更新 2026-06-02）
> 状态：草案 v0.1，待 owner review。
> 关联：`ml_pv_clf_v0` runner、`scripts/strategy1/render_report.py`、`PRD_20260601_02_策略1BQML回测闭环.md`、`PRD_20260602_02_OQ010交易成本口径.md`

---

## 1. 背景

策略 1 BigQuery ML runner 已能端到端产出 ADS 回测结果和基础报告，但当前报告仍停留在技术验收级别：

- 报告标题、指标名、图表名仍为英文。
- 报告正文只展示汇总绩效、模型指标和 NAV / 回撤图，不能回答“买了什么、什么时候买卖、亏在哪里”。
- 当前 runner 示例基准仍为 `000852.SH`，而 owner 要求策略报告主基准改为沪深 300。
- 若策略效果不好，当前报告只能看到结果差，不能定位是模型选股差、某个回撤窗口集中亏损、交易成本吞噬收益、不可交易跳过，还是风格基准暴露问题。

本 PRD 定义策略 1 报告增强需求：**中文可读、沪深 300 主基准、交易明细可追溯、亏损窗口可解释、AI 诊断基于结构化证据**。

## 2. 目标

1. 报告正文使用中文书写，面向 owner / 策略研究者直接阅读。
2. 默认主基准改为沪深 300 canonical `sec_code='000300.SH'`。
3. 报告中体现策略买卖细节：正文展示摘要表，完整成交 / 持仓 / NAV 明细作为附件输出。
4. 当策略表现不好时，自动生成失败定位证据，并由 AI 基于证据写中文分析。
5. 报告仍保持可复现：所有结论可追溯到 ADS 表、运行参数、artifact 文件和生成时间。

## 3. 非目标

- 不在本 PRD 中改模型、特征、标签、持仓数或调仓频率。
- 不把新闻、公告、研报等外部文本原因纳入 P0；没有外部证据时，AI 不得编造“公司新闻/行业事件原因”。
- 不把完整逐笔明细全部塞进 HTML 正文；完整明细以 CSV / JSON 附件形式保存。
- 不新增通用 AI 平台或长期 prompt 管理系统；本期只服务策略 1 报告。
- 不在仓库中保存 API key、OAuth token、BigQuery key 或任何 LLM 凭据。

## 4. 使用者与核心问题

| 使用者 | 需要回答的问题 |
|---|---|
| owner | 策略是否跑赢沪深 300？亏损是不是集中在某几个时间点？买了哪些票导致亏损？ |
| 策略研究者 | 模型打分、选股、持仓、成交、成本和不可交易约束分别贡献了什么？ |
| 工程维护者 | 报告是否由 ADS 事实表生成？本地 / GCS artifact 是否完整？是否可重复渲染？ |

## 5. 关键决策

### 5.1 主基准

P0 报告和 runner 默认主基准为沪深 300：

```text
benchmark_sec_code = '000300.SH'
benchmark_name = '沪深300'
```

要求：

- `08_run_backtest.sql` 和 `09_build_metrics_and_report_inputs.sql` 的 `p_benchmark` 默认值后续实现时改为 `000300.SH`。
- 仍必须执行 OQ-004 已有前置校验：`dim_index.has_daily=TRUE AND is_benchmark_candidate=TRUE`，且 NAV 窗口内 `dwd_index_eod` 对每个开市日有且只有一条有效价格记录。
- `ads_backtest_nav_daily.benchmark_sec_code`、`ads_backtest_performance_summary.benchmark_sec_code` 写主基准 `000300.SH`。

### 5.2 辅助风格基准

策略 1 原设计偏中小盘股票池；只看沪深 300 可能混淆“选股 alpha”与“风格 beta”。因此 P0 报告正文应保留辅助风格基准展示：

| 基准 | canonical `sec_code` | 用途 |
|---|---|---|
| 沪深 300 | `000300.SH` | 主基准，owner 默认对比口径。 |
| 中证 1000 | `000852.SH` | 辅助风格基准，用于判断中小盘 beta 暴露。 |
| 中证 500 | `000905.SH` | 可用时展示，作为中盘风格对照。 |

辅助基准不要求写入现有 ADS 主字段；报告渲染脚本可从 `ashare_dwd.dwd_index_eod` 按窗口查询，生成辅助基准曲线和相对收益。辅助基准若未通过 OQ-004 可用性校验，应在报告中写明“该辅助基准不可用”，不能静默缺失。

### 5.3 报告语言

P0 报告的用户可见文本必须为中文，包括：

- 标题、章节名、指标名、表头、图表标题、图例、脚注。
- 对 `run_id`、`model_id`、`backtest_id`、`sec_code` 等技术 ID 保留原值。
- 附件文件名可用英文稳定命名；报告正文需提供中文附件说明。

### 5.4 AI 分析边界

AI 只能基于报告证据包分析，不得凭空补充未提供的数据源。

必须区分：

- **事实**：来自 ADS / DWD / DWS / artifact 的日期、股票、收益、成本、回撤、成交、持仓、分数、rank。
- **推断**：基于特征、持仓和行情路径做出的解释，例如“亏损主要集中在高波动小市值持仓”。
- **不可判断**：当前数据不含新闻、公告、行业突发事件时，不得写成外部事件原因。

## 6. 触发条件

报告每次都生成“风险与归因”章节；当满足以下任一条件时，必须生成“AI 诊断”章节：

| 条件 | 默认阈值 |
|---|---:|
| 累计收益小于沪深 300 累计收益 | 策略累计收益 < 主基准累计收益 |
| 策略累计收益为负 | `total_return < 0` |
| 最大回撤较大 | `max_drawdown <= -0.15` |
| 快速亏损窗口 | 任一 5 / 10 / 20 交易日滚动收益 `<= -0.08` |
| 成本侵蚀明显 | 总交易成本 / 期末净资产收益绝对值 `>= 0.2` |

阈值必须参数化，并写入 `metrics_json.report_diagnosis_config`。

## 8. 报告正文结构

### 8.1 首页摘要

必须包含：

- 策略名称、`run_id`、`backtest_id`、`model_id`、生成时间。
- 主基准：沪深 300 `000300.SH`。
- 训练 / 验证 / 测试 / 回测窗口。
- 初始资金、成本 profile、调仓频率、目标持股数、单票权重上限。
- 一句话结论：跑赢 / 跑输沪深 300、主要风险来源、是否触发 AI 诊断。

### 8.2 绩效总览

至少展示：

- 累计收益、年化收益、年化波动、Sharpe、最大回撤、Calmar。
- 沪深 300 累计收益、累计超额收益、信息比率。
- 辅助风格基准累计收益和超额收益。
- 年化换手、总成本、成本占收益比例。
- 买入 / 卖出不可交易跳过次数和比例。

### 8.3 图表

至少生成：

1. 策略净值 vs 沪深 300 vs 辅助风格基准。
2. 策略回撤曲线。
3. 相对沪深 300 超额收益曲线。
4. 换手与交易成本曲线。

图表标题、图例、坐标轴均为中文。

### 8.4 买卖细节

正文展示摘要表，附件保存完整明细。

正文至少展示：

- 最近 20 笔成交。
- 成交额最大的 20 笔成交。
- 亏损贡献最大的 20 个持仓 / 股票。
- 不可交易跳过的 BUY / SELL 意图行样例。

完整 `trades.csv` 至少包含：

| 字段 | 说明 |
|---|---|
| `trade_date` | 成交 / 意图日期 |
| `sec_code` | 证券代码 |
| `side` | BUY / SELL |
| `planned_shares` | 计划股数 |
| `filled_shares` | 实际成交股数 |
| `fill_price` | 成交价 |
| `turnover_cny` | 成交额 |
| `fee_cny` | 佣金 |
| `tax_cny` | 印花税 |
| `slippage_cny` | 滑点成本 |
| `cash_effect_cny` | 现金影响 |
| `fill_status` | 成交状态 |
| `score` | 买入对应模型分数，能 join 到时填写 |
| `rank_raw` | 买入对应模型排名，能 join 到时填写 |
| `target_weight` | 目标权重 |
| `position_weight_after` | 成交后最近持仓权重，能计算时填写 |

### 8.5 持仓与亏损归因

P0 不强制做复杂逐 lot 实现盈亏归因，采用持仓窗口贡献法：

```text
某股票在某窗口的贡献 ~= 窗口内每日持仓权重 × 股票当日收益 的累加
```

要求：

- 识别最大回撤窗口：峰值日、谷底日、回撤幅度、持续交易日数。
- 识别快速亏损窗口：5 / 10 / 20 交易日滚动亏损最大的窗口。
- 对每个关键窗口列出前 20 个亏损贡献股票。
- 对每个亏损贡献股票列出该窗口内相关买入 / 卖出记录、持仓权重峰值、买入时 score/rank。
- 若 `position_daily` 或价格数据不足，报告必须说明归因覆盖率，不能输出看似完整的结论。

### 8.6 AI 诊断

AI 诊断章节应按固定结构输出：

1. **结论摘要**：策略表现是否差，差在哪里。
2. **亏损最快的窗口**：日期、跌幅、相对沪深 300 表现。
3. **主要亏损股票**：列出股票、贡献、买入/持有/卖出时间。
4. **可能原因**：只基于证据包中的特征、分数、持仓、成本、不可交易、基准表现推断。
5. **不是原因 / 无法判断**：明确说明当前数据不足以判断新闻、公告、行业事件等外部原因。
6. **下一步改进建议**：例如降低换手、改标签、反转 score、调持股数、增加市场状态特征等。

AI 输出必须引用具体日期和证券代码；不得只写泛泛判断。

## 9. 数据读取与计算要求

### 9.1 现有 ADS 来源

| 来源表 | 用途 |

| `ads_portfolio_target_daily` | 调仓日目标权重。 |

### 9.2 DWD / DIM 来源

| 来源表 | 用途 |
|---|---|
| `dim_index` | 主/辅助基准可用性校验。 |
| `dwd_index_eod` | 辅助基准曲线。 |

所有大表读取必须带日期分区过滤。

### 9.3 证据包

`diagnosis_evidence.json` 必须至少包含：

- `run_context`：运行参数、模型、窗口、主/辅助基准、成本 profile。
- `performance_summary`：策略、沪深 300、辅助基准的收益风险指标。
- `drawdown_events`：最大回撤和 top N 回撤窗口。
- `fast_loss_events`：5 / 10 / 20 日快速亏损窗口。
- `loss_contributors`：每个窗口前 20 个亏损贡献股票。
- `trade_samples`：关键窗口内相关成交明细。
- `cost_breakdown`：佣金、印花税、滑点、总成本、换手。
- `execution_diagnostics`：不可交易跳过、现金、gross exposure。
- `model_signal_diagnostics`：关键股票买入时 score/rank、当日候选池规模、rank 分布。
- `data_coverage`：归因覆盖率和缺失说明。

## 10. AI 集成要求

### 10.1 运行模式

报告脚本新增参数：

```text
--ai-analysis-mode auto|off|evidence_only|llm
```

| 模式 | 行为 |
|---|---|
| `off` | 不生成 AI 诊断，只生成证据包。 |
| `evidence_only` | 不调用 LLM，用规则模板生成中文证据摘要。 |
| `llm` | 必须调用 LLM；失败则报告生成失败。 |
| `auto` | 默认。触发失败条件且凭据可用时调用 LLM；否则退化为 `evidence_only` 并记录状态。 |

P0 默认建议为 `auto`，保证没有 LLM 凭据时报告仍可生成。

### 10.2 凭据与安全

- LLM 凭据只能通过环境变量或本机受信凭据提供。
- 不得写入仓库、报告、metrics、prompt 文件或日志。
- `ai_analysis.json` 只保存模型名、生成时间、输入 evidence hash、输出文本和状态，不保存 API key。

### 10.3 Prompt 约束

Prompt 必须要求模型：

- 用中文输出。
- 只引用 `diagnosis_evidence.json` 中存在的事实。
- 对没有证据的数据源写“当前证据不足，无法判断”。
- 列出关键日期、股票代码、窗口收益和贡献。
- 将“事实”和“推断”分段。

## 12. 验收标准

1. `report.md` 和 `report.html` 的用户可见内容为中文。
2. 默认主基准写入 `000300.SH`，报告显示为“沪深300”。
3. 报告至少输出 `trades.csv`、`positions.csv`、`nav.csv`、`drawdown_events.csv`、`loss_attribution.csv`、`diagnosis_evidence.json`。
4. `trades.csv` 行数与同 `backtest_id` 的 `ads_backtest_trade_daily` 行数一致。
5. 报告正文展示最近成交、最大成交、亏损贡献股票和不可交易跳过样例。
6. 若触发失败条件，报告包含“AI 诊断”或“证据摘要”章节，且列出具体日期和股票代码。
7. AI 诊断不得出现证据包没有的数据源原因；如无新闻/公告数据，必须明确“当前证据不足”。
8. `metrics_json` 写入 `report_version`、`artifact_manifest`、`ai_analysis_status`、`diagnosis_triggered`。
9. GCS uploaded 模式仍只在真实上传成功时写 `report_uri`；local-only 模式不得写虚假的 `report_uri`。
10. `10_qa_runner_outputs.sql` 通过新增报告断言。

## 13. 风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| 中文报告和 CSV 明细过大 | HTML 难读、渲染慢 | 正文只展示 top N，完整明细放附件。 |
| AI 编造外部原因 | 误导策略判断 | 证据包约束 + prompt 明确禁止 + 报告标注“外部原因未接入”。 |
| 亏损归因用 lot 匹配复杂 | P0 实现变大 | P0 用持仓窗口贡献法，P1 再做逐 lot。 |
| 沪深 300 主基准与小盘策略风格不匹配 | 错判 alpha | 同时展示中证 1000 / 中证 500 辅助风格基准。 |
| 无 LLM 凭据导致报告失败 | 阻塞 runner QA | `auto` 模式退化为 `evidence_only`，仍产出可读证据摘要。 |
