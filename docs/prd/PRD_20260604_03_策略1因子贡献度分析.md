# PRD: 策略 1 因子贡献度分析

## 1. 背景

策略 1 当前正式 baseline 已跑通：

- `run_id`: `s1_bqml_baseline_pvfq_n30_bw_h5_v20260604_01`
- `backtest_id`: `bt_s1_bqml_baseline_pvfq_n30_bw_h5_v20260604_01`
- 参数: `pv_fin_quality + 30/5% + biweekly + 5d`
- 回测区间: `2024-01-02` 至 `2025-12-31`
- 主基准: `000852.SH`
- 结果: total_return=41.10%, excess_return=12.09%, Sharpe=1.043, max_drawdown=-14.48%

现有诊断已经能回答“信号是否可用”“高低分 bucket 是否有效”“组合是否过度集中”“成本换手是否异常”等问题，但还不能系统回答：

1. 模型主要依赖哪些因子。
2. 哪些因子方向与未来收益一致。
3. 当前持仓暴露在哪些因子上。
4. 策略收益表现更像来自动量、估值、质量、规模、流动性，还是财务新鲜度 / 可用性。

本文定义一个不改变模型、不改变交易、不重新训练的因子贡献度分析模块。

## 2. 实施顺序

本文只定义实现顺序，不定义优先级。

推荐顺序：

1. 先实现本文的因子贡献度分析。
2. 再实现 `PRD_20260604_01_策略1LedgerV1交易执行语义.md` 的 Ledger v1 P0/P1/P2。
3. 最后实现 `PRD_20260604_02_策略1月度滚动重训.md`。

理由：

1. 因子贡献度分析只读已有 model / prediction / backtest / feature 数据，不改变交易执行语义。
2. 先看清当前 baseline 的因子来源，有助于后续解释 Ledger v1 后收益变化是否只是执行语义变化，而不是模型本身暴露异常。
3. 因子贡献度产物可以复用于 Ledger P0/P1/P2 和月度重训后的结果对比。

## 3. 目标

1. 输出当前 selected BQML 模型的因子系数、标准化系数和 orientation 后方向。
2. 输出每个因子的 valid/test RankIC、bucket lift 和 top-minus-bottom 收益关联。
3. 输出模型分数层面的因子贡献：全预测池、高分组、候选池、实际持仓的平均 score contribution。
4. 输出组合持仓层面的因子暴露：实际持仓相对全市场预测池的加权 z-score 暴露。
5. 输出近似的组合因子收益关联：持仓因子暴露 × 日截面单因子收益斜率。
6. 将结果写入 GCS / 本地镜像，并回写 ADS summary `metrics_json`。
7. 在中文报告或模型诊断报告中展示因子贡献摘要。

## 4. 非目标

1. 不做消融实验。
2. 不 drop-one-factor / drop-one-factor-group 重训。
3. 不改变模型、score orientation、预测池、组合参数、成本参数或回测区间。
4. 不把因子贡献解释为严格因果。
5. 不引入 SHAP / permutation importance 等需要大量重复预测的解释方法。
6. 不新增生产交易规则。

## 5. 分析口径

### 5.1 三类贡献

本文把“因子贡献度”拆成三类，避免混用概念。

| 类型 | 回答的问题 | 产物 |
|---|---|---|
| 模型分数贡献 | 模型为什么给这只股票高分？ | `factor_model_weights.csv`, `factor_score_contribution_*.csv` |
| 预测收益关联 | 单个因子自身是否与未来收益同向？ | `factor_rank_ic_*.csv`, `factor_bucket_lift_*.csv` |
| 组合暴露 / 近似归因 | 实际持仓偏向哪些因子，这些因子在该窗口是否赚钱？ | `portfolio_factor_exposure_daily.csv`, `portfolio_factor_attribution_proxy.csv` |

### 5.2 不是消融实验

本文所有结果均基于已训练模型和已生成预测，不重新训练、不重跑参数网格。

允许做：

1. `ML.WEIGHTS` 读取 selected BQML 模型系数。
2. 使用现有训练面板 / 预测池 / 持仓表计算因子 z-score、RankIC、bucket lift。
3. 使用单因子日截面斜率做组合归因 proxy。

禁止做：

1. 重新训练不含某因子的模型。
2. 重新训练不含某因子组的模型。
3. 用消融结果更新默认参数。

## 8. 计算规则

### 8.1 模型系数与标准化系数

对 selected model 执行 `ML.WEIGHTS`，输出：

1. `raw_coefficient`。
2. `feature_mean_train`、`feature_std_train`。
3. `standardized_coefficient = raw_coefficient * feature_std_train`。
4. `oriented_coefficient`：
   - `score_orientation='identity'`: 等于 `raw_coefficient`。
   - `score_orientation='reverse_probability'`: 方向取反，用 `-raw_coefficient` 表示最终排序分的边际方向。
5. `oriented_standardized_coefficient` 同理取向后标准化。

说明：

1. 当前 `preprocess_version='raw_v0'`，原始系数不可直接按绝对值比较。
2. 排名默认看 `ABS(oriented_standardized_coefficient)`。
3. 截距只保留在模型系数文件中，不参与因子排名。

### 8.2 单因子 RankIC

对每个 `trade_date`、每个因子，在 label-available eval subset 上计算 Spearman RankIC：

```text
RankIC(feature, target_return)
```

要求：

1. valid/test 分开输出。
2. `target_return` 必须与 `p_label_horizon` 对齐。
3. 只用事后标签成熟样本评价，不能用标签有效性过滤 live prediction pool。
4. 每个日截面有效样本数低于阈值时跳过该日，默认阈值 `min_daily_factor_samples=100`。

### 8.3 单因子 bucket lift

每个交易日按单因子值做 5 bucket 分层，计算：

1. bucket 1/2/3/4/5 平均未来收益。
2. `top_minus_bottom = bucket_5 - bucket_1`。
3. `direction_adjusted_top_minus_bottom`：
   - 若因子方向为正，等于 `bucket_5 - bucket_1`。
   - 若因子方向为负，等于 `bucket_1 - bucket_5`。

因子方向优先用 `oriented_standardized_coefficient` 确定；若模型未使用该因子，则用 valid RankIC 均值符号。

### 8.4 模型分数贡献

对每条预测记录和每个因子计算：

```text
raw_score_contribution = feature_value * raw_coefficient
oriented_score_contribution =
  raw_score_contribution                       if score_orientation = identity
  -raw_score_contribution                      if score_orientation = reverse_probability
```

输出按以下 group 聚合的均值、标准差、p10/p50/p90：

1. `all_predictions`
2. `top_30pct`
3. `bottom_30pct`
4. `selected_candidate`
5. `held_position`

P0 不要求把每只股票 × 每日 × 每因子的明细全部落本地文件；可在 BigQuery 聚合后只输出 summary，避免文件过大。

### 8.5 组合因子暴露

对每个交易日、每个因子：

1. 在当日 prediction pool 上做横截面 z-score。
2. 计算组合持仓加权暴露：

```text
portfolio_factor_exposure = SUM(position_weight * factor_zscore)
```

3. 同时输出 selected candidate 的等权暴露，便于区分“选股暴露”和“成交后持仓暴露”。

### 8.6 组合因子归因 proxy

P0 使用单因子日截面斜率做近似归因：

```text
factor_return_proxy_d = slope(target_return_d ~ factor_zscore_d)
portfolio_factor_proxy_contribution_d =
  portfolio_factor_exposure_d * factor_return_proxy_d
```

注意：

1. 这是收益关联 proxy，不是严格 P&L 分解。
2. 不允许在报告中写成“该因子导致收益”。
3. 报告用语必须是“贡献度 proxy”“收益关联”“暴露收益估计”。
4. 实际个股 P&L 仍以现有 `loss_attribution.csv` / 个股贡献表为准。
