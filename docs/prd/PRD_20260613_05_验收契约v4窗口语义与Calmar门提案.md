## 1. 问题（PR #211 已证实）

v3 `absolute_performance_gate`（Sharpe ≥ 0.70 且 Calmar > 1.0）**没有钉死评估窗口**：五轮历史 search 的 replay 在 `2024-01-02..2026-04-30` 近窗执行（并已产生 1 accepted / 24 rejected），而当前 annual-rolling continuous baseline 一直按 2021 起长窗叙事评估。同一个 baseline：近窗 Sharpe `1.46` / Calmar `1.10` 双门皆过，长窗 `0.6685` / `0.4103` 双门皆败。门的实际含义由窗口决定，这是契约缺陷。

## 2. v4 提案（我的具体建议）

### 2.1 窗口语义：双窗分层，各司其职

- **验收窗（gates 适用）**= 滚动近窗：**最近两个完整自然年 + 当年 YTD**。机器可执行定义（review 后补齐）：
  - `evaluation_end_date` 解析顺序 = `min(被评估 backtest 的 predict_end, CA 事件可见上界, NAV 与全部 comparison benchmark 的共同完整截止日)`，三者任一缺失即 fail-closed；
  - `evaluation_start_date` = `evaluation_end_date` 往前数第二个完整自然年的 1 月首个开市日（当前评估即 `2024-01-02`）；
  - **YTD 最小样本**：当年 YTD 段不足 **40 个交易日**（对齐既有 `final_holdout_gate.trading_day_count >= 40` 口径）时，验收窗退化为"最近两个完整自然年"、不含 YTD，并在 artifact 记录退化标记；
  - 滚动：每个自然年度首个评估周期整体前移一年。
  理由：验收回答"现在部署它好不好"，近窗贴近部署分布；v3 阈值当初在 `2024-01-02..2026-04-30` 校准，保持判定连续性；该窗口下 Calmar > 1.0 已被 baseline 与历史候选证明可达——"不可达"问题随窗口钉死而消失。
- **长窗诊断义务（不设硬门）**：任何 acceptance artifact 必须披露自 2021-01-04 起（或策略可得最早日）的长窗 CAGR / Sharpe / Calmar / MaxDD 与 crunch 段表现；**长窗 MaxDD 深于 `-45%` 时，production-accept 必须 owner 显式 sign-off**（软门：不自动拒绝，但不许静默通过）。理由：保留两轮 -40% 回撤期的信息量，又不把市场结构锁死的分母变成永久不可达硬门。

### 2.2 Calmar 门：窗口钉死后维持阈值，不降、不改超额口径

- 近窗 `calmar_ratio > 1.0` 与 `sharpe_ratio >= 0.70` **保留原值**——窗口钉死后该组合已被证明可达且有区分度（25 个历史候选仅 1 个过全门）。
- 不采纳选项 A（降阈值：近窗下指数 beta 即可过 0.5 量级，区分度崩塌）、不采纳选项 B（超额 Calmar：同窗超额回撤分母语义复杂且可博弈，PR #211 §2.4 已列证据）、不采纳裸选项 C（删绝对 Calmar：近窗 6/8 纯指数可过其余组合，防 beta 完全压到 Sharpe+relative 两道门上，冗余度不足）。

### 2.3 防 beta 双护栏（v4 新增）

1. **pure-index guard 固化为契约 QA**——机器语义（review 后补齐）：指数没有 signal/CV 字段，guard 只对**可计算子门组合**求值 = `absolute_performance_gate`（Sharpe/Calmar，指数 NAV 取 `dwd_index_eod` close 归一序列）+ `relative_gate`（指数互为 benchmark，**排除 self-benchmark**）；断言**没有任何指数能同时通过该子门组合**。若有 → 判 gate 设计失效，**中止本次 acceptance 评估并报 owner**（不放行候选、不静默降级）；任一指数同窗数据不完整 → fail-closed。这正是把 PR #211 的底线检查变成常设机制：防的是"beta 即可过 performance+relative 组合"，signal 类门对指数本就不可算、不计入 guard。
2. `signal_quality_gate`（valid/test rank IC > 0 等）维持必过——beta 类标的没有信号证据，结构上进不了正门；guard 与 signal gate 双层互补。

### 2.4 分级（吸收选项 D 的合理部分）

- **research-accepted** = 验收窗全门通过 + 长窗诊断披露完成。允许据此切换研究 baseline、排实验优先级。
- **production-accepted** = research-accepted + final holdout 无明显坏结果 veto + 长窗 MaxDD 软门处置完毕 + **owner 显式批准 promotion**（现行 promotion 纪律原样保留，分级不放松任何一道既有人工闸门）。

## 3. 后果反事实（owner 批准前必须知情的部分）

- **最大后果**：当前 true5y CA-on baseline 在 v4 口径下近窗绝对门**会过**（Sharpe 1.46 / Calmar 1.10）；其是否达到 research-accepted 还取决于 signal_quality / relative gate / pure-index guard 在同窗的实跑判定（本 PRD 不预判）。若全过，它将成为首个 research-accepted 候选——**这意味着 OQ-010 的"寻找可 accepted baseline"主线可能就此改变性质**（从"找到能过门的策略"变为"对已过门 baseline 做 production 闸门与增强"）。owner 批准 v4 即接受这一可能性。
- 历史五轮 search 的 25 个 canonical 候选在 v4 下的判定矩阵：实施阶段逐一反事实回放（复用 PR #211 脚本与数据契约），结果入 v4 切换报告；预期与 v3 近似（窗口未变，新增的是 guard 与分级），已 accepted 的 1 个候选需重过 pure-index guard。
- 长窗 MaxDD 软门对当前 baseline 的含义：`-37.43%` 浅于 `-45%` 阈值，不触发 sign-off 要求。
- **与 PRD_20260613_04 Phase 2 的判读边界**：Phase 2 topdown 结果的判读主体始终是其预登记的**构造对比**（construction evidence，长窗 vs baseline）；v4 若获批，只**附加**一张 v4 shadow 判定表（对已落库 NAV 只读复算近窗 gates），不改变 Phase 2 预登记判读，也不因 shadow 结果触发任何 accepted 标记。
- **resume / stitched segment 边界**：acceptance 评估对象只能是 full continuous ledger 或 owner 显式批准的 stitched continuous artifact（如 dividend 修正后的 parent+child 拼接口径整体）；`bt_s1_dividend_backfill_resume_*` 这类 child segment **不得**作为独立 acceptance candidate，也不得单独充当 YTD/验收窗数据源。
