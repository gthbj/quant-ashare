# PRD：OQ-006 接口单位换算口径与准入门禁

> 文档维护：GPT-5（最近更新 2026-06-02）
> 状态：草案 v0.4，owner 关键决策已确认，已按 review 修订 index_daily 换算缺失。
> 范围声明：本文覆盖 OQ-006 的单位契约、机器可读映射、覆盖检查、DWD 落库准入门禁、P0 已落库字段补契约要求，以及 PR #13 财务三表落地暴露的单位追溯要求；不直接实现 BigQuery meta 表、QA SQL 或改造现有 DWD 字段。
> 上游依据：`docs/数据仓库建模方案-DWD-DIM.md` §3.3-G、§10；`.agent/memory/OPEN_QUESTIONS.md` OQ-006；`.agent/memory/KNOWN_CONSTRAINTS.md` Tushare 单位约束。

## 1. 背景

OQ-006 当前问题不是“缺一张人工备忘表”，而是缺一套可审计、可执行的单位准入机制。

Tushare 不同 endpoint 的同名字段可能单位不同。例如股票 `daily_basic.total_mv` 与指数 `index_dailybasic.total_mv` 不能按字段名统一乘同一个系数；`amount`、`vol`、`share`、`mv` 等字段也必须按 `endpoint + source_field` 判断来源单位。

当前项目已在 P0 表里做了若干单位归一：

- `daily.vol`：手 -> 股。
- `daily.amount`：千元 -> 元。
- `daily_basic` 股本字段：万股 -> 股。
- `daily_basic` 市值字段：万元 -> 元。
- `index_dailybasic` 已确认不同于股票 `daily_basic`，市值/股本按元/股使用。
- `index_daily.vol` / `index_daily.amount` 当前在 `dwd_index_eod` 中仅直接 `SAFE_CAST` 为 `volume` / `amount`，仍是源单位手 / 千元；OQ-006 实现必须补 `*100` / `*1000` 换算并同步命名迁移。

但这些规则仍主要分散在 SQL、字段描述和记忆里。PR #13 财务三表以及后续 P1 资金流、融资融券、北向、龙虎榜、财务扩展、股东增减持、质押回购、指数权重等更多接口都会暴露金额、数量、价格或比率字段，如果继续依赖人记忆，最容易出现 1,000 倍、10,000 倍或百分比/比例混用错误。

本文建议把 OQ-006 收敛为：**单位契约 + 覆盖检查 + DWD 落库准入门禁**。

## 2. 问题定义

需要解决四类风险：

1. **同名字段误用**：只按 `amount` / `vol` / `total_mv` 字段名换算，忽略 endpoint 差异。
2. **DWS/ADS 二次换算**：DWD 已经标准化后，下游又乘 `1000` 或 `10000`。
3. **新表无单位证据**：新增 DWD 时没有官方文档证据或数据自洽校验，靠经验落库。
4. **历史口径不可审计**：P0 已落库字段虽然多数字段已正确换算，但缺少统一映射和 QA 门禁证明。

## 3. 目标与非目标

### 3.1 目标

1. 为所有进入 DWD 的金额、数量、股本、市值、价格、比率字段建立 `endpoint + source_field` 粒度的单位契约。
2. 对外 DWD/DWS/ADS 标准字段统一使用元、股或明确的百分数/比例口径。
3. 新 DWD 表含金额、数量、股本、市值、价格或比率字段时，必须先补单位映射和 QA，否则不得合并。
4. P0 已落库的 `daily`、`daily_basic`、`index_daily`、`index_dailybasic` 字段补齐单位契约记录。
5. PR #13 的 `income` / `balancesheet` / `cashflow` 若先于 OQ-006 合并，OQ-006 实现必须把三大财务表纳入首批补契约范围；若 OQ-006 先合并，则 PR #13 必须随表落地补单位映射。
6. 用 `sql/qa/05_oq006_unit_checks.sql` 检查映射覆盖、verification 状态、命名和算术自洽。

### 3.2 非目标

- 不在本文中一次性核完全部 57 张 ODS 表。
- 不把模糊字段强行标为 verified；证据不足时必须为 `pending`，且不能作为正式 DWD 标准字段。
- 不要求 DWS/ADS 维护独立单位表；DWS/ADS 只消费已标准化的 DWD 字段。
- 不在 PRD 阶段修改已物化 BigQuery 表结构；现有命名债务由后续实现 PR 处理。

## 6. 单位契约表设计

建议新增：

```text
data-aquarium.ashare_meta.ods_field_unit_map
```

字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `source_system` | STRING | 固定 `tushare` |
| `endpoint` | STRING | Tushare endpoint，如 `daily`、`daily_basic` |
| `ods_table` | STRING | ODS 表名 |
| `source_field` | STRING | ODS 原字段 |
| `semantic_type` | STRING | `amount` / `volume` / `share` / `market_value` / `ratio` / `price` |
| `source_unit` | STRING | 原单位，如 `千元`、`万元`、`手`、`万股` |
| `canonical_unit` | STRING | 标准单位，如 `元`、`股`、`元/股`、`percent` |
| `multiplier` | NUMERIC | ODS 原值乘以该系数得到 DWD 标准字段 |
| `dwd_table` | STRING | 消费它的 DWD 表 |
| `dwd_field` | STRING | DWD 标准字段 |
| `raw_field_kept` | BOOL | 是否保留源单位字段 |
| `raw_field_name` | STRING | 如 `amount_k_cny`、`total_mv_10k_cny`、`volume_lot` |
| `verification_status` | STRING | `verified` / `pending` / `not_applicable` |
| `naming_exception_type` | STRING | 命名例外类型，默认 NULL；可选 `legacy_unsuffixed`、`source_name_passthrough` |
| `naming_exception_expires_at` | DATE | 命名例外计划清理日期；无例外时为空 |
| `evidence` | STRING | 官方文档 URL、截图记录、实测校验说明或内部审计链接 |
| `verified_at` | DATE | 核对日期 |
| `verified_by` | STRING | 核对者或 Agent 模型名 |
| `notes` | STRING | 特殊口径说明 |

约束：

- 主键建议为 `(source_system, endpoint, ods_table, source_field, dwd_table, dwd_field)`。
- `verification_status='verified'` 时，`source_unit`、`canonical_unit`、`multiplier`、`evidence`、`verified_at` 均不能为空。
- `verification_status='pending'` 的字段不能作为当前 DWD 标准输出字段。
- `naming_exception_type` 只表示命名兼容例外，不替代 `verification_status`；例外字段仍必须完成单位核验。
- `naming_exception_type='legacy_unsuffixed'` 必须有 `naming_exception_expires_at` 或明确迁移 TODO；新 DWD 表不得新增该例外。
- 比率类字段不做元/股换算，但必须说明是百分数、比例还是 basis point。

比率字段的方向性规则：

- Tushare 源字段若官方口径为百分数，例如 `pct_chg`、`turnover_rate`、`dividend_yield`，DWD 可 pass-through 保留源口径，`canonical_unit='percent'`，字段描述需说明“5.2 表示 5.2%”。
- DWD 自算收益率 / 比例字段统一使用 ratio，例如 `ret_1d=0.052` 表示 5.2%，`canonical_unit='ratio'`。
- 估值倍数或流动性倍数字段按 `multiple` 登记，例如 `pe`、`pb`、`ps`、`volume_ratio`。

### 7.3 P0 命名债务 + 换算缺失

现有 `dwd_index_eod.volume` 与 `dwd_index_eod.amount` 未带标准单位后缀，且当前 SQL 直接 `SAFE_CAST(d.vol)` / `SAFE_CAST(d.amount)`，实际落库仍是源单位“手” / “千元”，不是 DWD 标准单位“股” / “元”。owner 已确认：OQ-006 实现 PR 必须同时修复换算和命名，不接受长期登记 legacy exception 作为关闭方案。

实施要求：

1. `dwd_index_eod` 标准输出字段必须改为 `volume_share = SAFE_CAST(index_daily.vol AS FLOAT64) * 100.0`、`amount_cny = SAFE_CAST(index_daily.amount AS FLOAT64) * 1000.0`。
2. 建议同步保留 raw 字段 `volume_lot` / `amount_k_cny`，用于审计和自洽 QA；如不保留 raw 字段，单位契约也必须记录源字段和 multiplier。
3. 更新下游引用、metadata 和 QA，并重建 `dwd_index_eod`。
4. 如为了兼容旧 SQL 需要短期保留 `volume` / `amount`，只能在兼容 view 或明确标注的过渡字段中保留，不得作为 DWD 标准字段。
5. 临时例外必须写 `naming_exception_expires_at` 或明确迁移 TODO；`verification_status` 仍必须为 `verified`。
6. 新 DWD 表不得新增未带单位后缀的标准字段；`legacy_unsuffixed` exception 不得用于规避新表命名规范。例外：`source_name_passthrough` 适用于 `source_unit = canonical_unit` 且 `multiplier = 1` 的 pass-through 字段（源单位已为元/股/percent 等标准单位，无需换算），允许保留 Tushare 源字段名，不受本条限制。

## 8. P1 接入范围

OQ-006 最小实现必须先于 P1 资金流、财务扩展等高单位风险 DWD 正式落地。P1 每新增一个 DWD 表前，必须先补 endpoint 单位映射，并跑 `sql/qa/05_oq006_unit_checks.sql`。

| 类别 | 典型 endpoint | 重点字段 |
|---|---|---|
| 资金流 | `moneyflow`、`moneyflow_hsgt` | 买入/卖出/净流入金额、成交额占比 |
| 融资融券 | `margin`、`margin_detail` | 融资余额、融券余量、买入额、偿还额 |
| 北向/持股 | `hk_hold`、`ccass_hold` | 持股数、市值、占比 |
| 涨跌停/龙虎榜 | `limit_list_d`、龙虎榜相关 endpoint | 成交额、净买额、席位金额 |
| 股东/增减持 | holder trade / pledge / repurchase 类 endpoint | 股数、金额、比例 |
| 财务扩展 | 三大报表新增单季派生、其他财务 endpoint | 新增财务金额、每股指标、比率 |
| 行业/指数权重 | index weight / index member 类 endpoint | 权重、股数、金额、市值 |

## 9. 核对方法

每个字段必须走两步。

### 9.1 官方文档确认

- 查 Tushare endpoint 文档的字段说明和单位。
- `evidence` 必须写 URL、截图记录、导出文档位置或其他可复核证据。
- 文档未标单位或口径模糊时，`verification_status='pending'`。
- `pending` 字段不能落为 DWD 标准字段；如必须暂存，只能保留 raw 字段并标注不可用于下游正式特征。

### 9.2 数据自洽校验

| 字段类型 | 校验方式 |
|---|---|
| 成交额 | `amount_cny ~= close * volume_share`，允许行情聚合误差 |
| 总市值 | `total_mv_cny ~= close * total_share` |
| 流通市值 | `circ_mv_cny ~= close * float_share` |
| 指数市值 | `total_mv_cny` / `float_mv_cny` 与指数股本字段量级一致 |
| 资金流金额 | 买入、卖出、净流入与当日成交额量级合理 |
| 股本 | 日频不应异常跳变，拆股/送转附近除外 |
| 财务金额 | 与总资产、营收、净利润历史量级一致，不应差 1e4 或 1e8 |
| 比率字段 | 不做元/股换算，但检查是百分数、比例还是 bps |

## 10. DWD 落库规则

### FR-UNIT-1：标准字段必须带单位后缀

金额字段统一 `_cny`，数量/股本字段统一 `_share`，保留源单位字段时用 `_k_cny`、`_10k_cny`、`_lot`、`_10k_share` 等后缀。

例外：`source_name_passthrough` 适用于 `source_unit = canonical_unit` 且 `multiplier = 1` 的 pass-through 字段（如 Tushare 财报金额字段已为元，无需换算），允许保留源字段名，不受「新 DWD 表不得新增未带单位后缀」的限制；`legacy_unsuffixed` 仅用于历史已有字段，必须登记 `naming_exception_expires_at` 清理日期。

示例：

```text
daily.amount       -> amount_k_cny   -- raw，千元
daily.amount *1000 -> amount_cny     -- standard，元
daily.vol          -> volume_lot     -- raw，手
daily.vol *100     -> volume_share   -- standard，股
```

### FR-UNIT-2：换算只能发生在 ODS -> DWD

DWS/ADS 不允许出现单位乘法常数，例如 `* 1000`、`* 10000`。如确需派生金额或股数，必须基于 DWD 标准字段计算。

### FR-UNIT-3：高风险字段保留 raw 字段

资金流、财务、北向、持股、质押、回购等高风险接口，首版 DWD 建议同时保留 raw 字段和标准字段。

### FR-UNIT-4：字段 description 写清换算来源

字段描述必须写清源字段、源单位、换算系数和标准单位。

示例：

```text
成交额，元，由 Tushare daily.amount 千元 * 1000 换算
```

### FR-UNIT-5：新 DWD PR 必须附单位映射

新增或修改 DWD 标准字段时，PR 必须同时新增或更新 `ods_field_unit_map` 记录和 `05_oq006_unit_checks.sql` 覆盖范围，并把 `05_oq006_unit_checks.sql` 纳入该 PR 的必跑 QA。没有 verified 映射，不得合并。

### QA-UNIT-6：算术自洽检查

建议首版包含：

```text
daily:        amount_cny / NULLIF(close * volume_share, 0) 在合理分位区间内
index_daily:  amount_cny / NULLIF(close * volume_share, 0) 在合理分位区间内
daily_basic:  total_mv_cny / NULLIF(close * total_share, 0) 在合理分位区间内
daily_basic:  circ_mv_cny / NULLIF(close * float_share, 0) 在合理分位区间内
index_dailybasic: index total_mv_cny、total_share 与 dailybasic 单位量级不差 1e4
```

QA 不应因少量异常样本直接误杀；应使用分位数和异常样本输出。若出现系统性 1e3 / 1e4 偏差，直接失败。
