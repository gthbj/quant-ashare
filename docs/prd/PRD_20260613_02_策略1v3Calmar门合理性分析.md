## 1. 背景与问题

v3 `absolute_performance_gate` 要求 contract Sharpe ≥ 0.70 **且** Calmar > 1.0。问题的核心是**门的可达性是评估窗口依赖的**，现有证据按窗口分开陈述：

1. **2021-2026 长窗口（当前 annual-rolling continuous baseline 的评估口径）**：true5y CA-on baseline（dividend 修正后）Calmar = `0.4103`；该窗口含两轮 -40% 级指数回撤，long-only 策略 MaxDD 有 beta 下限，分母被市场结构锁死。
2. **2024-2026 短窗口（v3 replay 对五轮历史 search 的实际评估口径，`2024-01-02..2026-04-30`）**：历史候选**存在** Calmar > 1.0 的读数（如 LightGBM binary rank2 NAV 复算 Calmar ≈ 1.73、sklearn rank5 ≈ 1.53）——这些候选被拒是因其他 gate，**不能**把"历史候选全军覆没"当作 Calmar 门不可达的证据。两个窗口的差异本身（短窗避开 2021-2022 回撤）是分析必须正面处理的一等维度。
3. **NAV 级择时上界（旧口径证据，需重算）**：48 变体无摩擦 grid 最优 Calmar `0.4077` 是基于**旧 effective-window backtest**（该报告中对应 baseline Calmar `0.2646`）的 NAV，不是当前 CA-on baseline 的上界，不可与 `0.4103` 直接比较；本 PRD 需在当前 true5y CA-on NAV 上只读重算该上界（见 §2.3）。

同时一个**契约现状**需要厘清：v3 已存在 `relative_gate`（`excess_calmar_ratio > 1.0` 或 `strategy_max_drawdown_same_window_excess > 0` 任一分支 + 超额 CAGR > 0，any_one_benchmark），但 `absolute_performance_gate` 仍是独立硬门。若绝对 Calmar 门物理不可达，则 relative_gate 的存在并不能解锁 accepted，所有 OQ-010 努力都在追一个不可达目标。

## 2. 分析内容

### 2.1 契约语义厘清（先于数据分析）

精确陈述 v3 各 gate 的组合逻辑（required / signal_quality / absolute / relative / final_holdout 是 AND 还是分支），引用 `acceptance.py` 实现与 contract 原文，确认：绝对 Calmar 门失败时 relative 分支是否有任何救济路径。结论写成一段事实陈述（这是后续选项设计的前提，且历史上 v2→v3 切换时的设计意图需从 PRD_20260608_02 与 DECISION-20260608-11/-12 中引证）。

### 2.2 同期指数 Calmar 对照（八指数，全量 benchmark 候选）

对 `dim_index` 中 `has_daily AND is_benchmark_candidate` 的全部 8 个指数（`000852.SH` 中证1000、`000300.SH` 沪深300、`000905.SH` 中证500、`000001.SH` 上证、`000016.SH` 上证50、`000688.SH` 科创50、`399001.SZ` 深证成指、`399006.SZ` 创业板指）：

1. **双窗口**计算：长窗（2021-01-04..2026-06-09，baseline 评估口径）与 v3 replay 窗（2024-01-02..2026-04-30）各算一遍 compound CAGR、MaxDD、Calmar、contract Sharpe（按 v3 metric_definitions 公式）；
2. 滚动窗口视角：3 年滚动 Calmar 的时间序列（评估"换个起点能不能过门"——若全部滚动窗口都远低于 1.0，不可达结论更稳；若短窗能过，量化窗口选择对门的影响）；
3. 加一行"持有 `000852.SH` 满仓不动"的策略视角读数（双窗口），和 baseline / 重算后的暴露仿真上界并排。

### 2.3 可达性证据汇总（含上界重算）

1. **重算 NAV 择时上界**：复用 `scripts/strategy1/simulate_exposure_overlay_upper_bound.py` 的只读路径，把输入 NAV 换成当前 true5y CA-on baseline（`bt_..._v20260611_01_ca01`，含 dividend resume 修正段拼接口径——直接用 official NAV 即可，resume 修正已在 #205 报告中证明对全期指标影响 <0.04pp），产出当前口径的无摩擦最优 Calmar 上界（沿用原 48 变体 grid 与 in-sample bias 声明）。
2. 把以下读数排成"Calmar 可达性阶梯"表（**按窗口分列**）：8 指数 Calmar 区间 → 满仓持有 000852 → 当前 baseline `0.4103`（长窗）→ 重算后的无摩擦择时上界 → 历史候选短窗最优读数（≈1.7，标注窗口差异）→ v3 门 `1.0`。回答两个问题：(a) 长窗口径下 Calmar > 1.0 需要什么数量级的结构改变（对冲 / 多空 / 跨域）；(b) 门的"不可达"有多少来自窗口选择而非策略能力。

### 2.4 备选门方案分析（只列选项 + 各自证据，不做推荐之外的裁决）

至少覆盖四个选项，每个给出"按该口径当前 baseline 与历史候选会怎样判定"的反事实回放（只读复算，不改任何 registry）。**反事实回放数据契约**（实现硬要求）：

- 历史候选源：`ashare_ads.ads_model_registry`（`status='selected'`，五个 replay search_id 按 v3 contract `replay_scope`）+ `ads_backtest_performance_summary` + `ads_backtest_nav_daily`；当前 baseline 源：research backtest NAV。
- 历史 ADS summary 的 `compound_annual_return` / v3 JSON 字段对旧 run 为空——**Calmar/Sharpe/CAGR 一律从 NAV 复算**（按 v3 metric_definitions 公式），不依赖 summary 字段。
- 一对多 backtest 去重规则：同一 model_id 可能 join 到多个 backtest（如 riskfeat reg rank4 有 4 个 portfolio 变体）；主口径按 `shortlist_rank_valid_only` 取模型 Top5 后，只取该 search 的 canonical 默认组合 backtest（`bt_s1_<search_id>__<candidate_id>` 形态），portfolio 变体（`n10/n20/n40` 后缀）单列为附表不入主判定；既有 `replay_acceptance_gate_v3.py` 的 `ensure_expected_top_k` 五条校验语义在本脚本中按上述规则适配（不修改 replay 脚本本身）。
- 反事实矩阵必须同时展示每个候选在**全部 gate**（required / signal_quality / absolute / relative）下的实际通过/失败原因，避免"只看 Calmar"误读。

四个选项：

- **A. 降低绝对 Calmar 阈值**：给出按指数对照与 baseline 分布推导的候选阈值（如同期最优指数 Calmar 的 1.5×，或 0.5 这类整数锚），列出该阈值下历史候选的判定变化；
- **B. 绝对门改超额口径**：absolute gate 的 Calmar 替换为已有 metric_definitions 中的 `excess_calmar_ratio`（vs 主基准 000852.SH），分析该指标的分母语义（同窗超额回撤）与可博弈性；
- **C. 双轨门**：绝对 Sharpe 门保留，Calmar 从 absolute gate 移到 relative gate 分支（实质 = 删除绝对 Calmar 硬门，靠既有 relative_gate 兜底），分析放权后的风险（纯 beta 策略是否会漏过——用 8 指数自身过门情况验证）；
- **D. 不改门**：保留 1.0 作为"北极星"，accepted 标准另行分级（research-accepted / production-accepted），分析对 OQ-010 路线优先级的含义。

每个选项必须回答：该口径下"满仓持有指数"会不会被误放行（门的反 beta 滥竽充数能力是底线）。

## 3. 预登记判读规则（先写后跑）

按窗口分层判读：

1. **长窗（2021-2026）不可达判定**：若 8 指数长窗 Calmar 全部 < 0.5 **且** 重算后的无摩擦择时上界 < 0.5 → "绝对 Calmar > 1.0 在长窗 long-only 域内物理不可达"成立。
2. **窗口敏感性判定**：若同时短窗（2024-2026）存在候选/指数/滚动窗口达到 1.0 量级（按 review 实查预期成立）→ 结论升级为"**门的实际含义由评估窗口决定，当前 contract 未钉死窗口口径是缺陷**"——owner 决策问题从"改阈值"扩展为"先钉死窗口语义（长窗 continuous vs replay 短窗），再定阈值/口径（选项 A-D）"。
3. 若长窗判定不成立（指数或上界 ≥ 0.5 量级）→ 结论为"门可达但当前策略族距离远"，改策略优先、门暂不动。
4. 本 PRD 不修改契约：任何门/窗口语义变更须另立 contract v4 / 修订 PRD 并经 owner 批准（DECISION-20260608-12 冻结纪律）。
