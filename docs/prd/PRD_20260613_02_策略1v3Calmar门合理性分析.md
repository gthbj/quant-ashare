## 1. 背景与问题

v3 `absolute_performance_gate` 要求 contract Sharpe ≥ 0.70 **且** Calmar > 1.0。三个事实指向"Calmar > 1.0 在 long-only 中证1000 域、2021-2026 窗口内可能物理不可达"：

1. 当前最优 baseline（true5y CA-on，dividend 修正后）Calmar = `0.4103`，历次全部候选（sklearn / LightGBM binary / regression / riskfeat 共 5 个 search 的 Top5）无一接近 1.0。
2. 暴露管理上限仿真：48 个变体 grid 中**无摩擦、含 in-sample selection bias** 的最优 NAV 级择时 Calmar 仅 `0.4077`——即使把 NAV 择时做到理论上限也距 1.0 很远，且该数字本身高估了样本外可达性。
3. 评估窗口 2021-01-04..2026-06-09 含两轮 -40% 级指数回撤；long-only 策略的 MaxDD 有 beta 下限，分母被市场结构锁死。

同时一个**契约现状**需要厘清：v3 已存在 `relative_gate`（`excess_calmar_ratio > 1.0` 或 `strategy_max_drawdown_same_window_excess > 0` 任一分支 + 超额 CAGR > 0，any_one_benchmark），但 `absolute_performance_gate` 仍是独立硬门。若绝对 Calmar 门物理不可达，则 relative_gate 的存在并不能解锁 accepted，所有 OQ-010 努力都在追一个不可达目标。

## 2. 分析内容

### 2.1 契约语义厘清（先于数据分析）

精确陈述 v3 各 gate 的组合逻辑（required / signal_quality / absolute / relative / final_holdout 是 AND 还是分支），引用 `acceptance.py` 实现与 contract 原文，确认：绝对 Calmar 门失败时 relative 分支是否有任何救济路径。结论写成一段事实陈述（这是后续选项设计的前提，且历史上 v2→v3 切换时的设计意图需从 PRD_20260608_02 与 DECISION-20260608-11/-12 中引证）。

### 2.2 同期指数 Calmar 对照（八指数，全量 benchmark 候选）

对 `dim_index` 中 `has_daily AND is_benchmark_candidate` 的全部 8 个指数（`000852.SH` 中证1000、`000300.SH` 沪深300、`000905.SH` 中证500、`000001.SH` 上证、`000016.SH` 上证50、`000688.SH` 科创50、`399001.SZ` 深证成指、`399006.SZ` 创业板指）：

1. 同窗口（2021-01-04..2026-06-09）按 v3 metric_definitions 公式计算 compound CAGR、MaxDD、Calmar、contract Sharpe；
2. 滚动窗口视角：3 年滚动 Calmar 的时间序列（评估"换个起点能不能过门"——若全部滚动窗口都远低于 1.0，不可达结论更稳）；
3. 加一行"持有 `000852.SH` 满仓不动"的策略视角读数，和 baseline / 暴露仿真上界并排。

### 2.3 可达性证据汇总

把以下读数排成一张"Calmar 可达性阶梯"表：8 指数同期 Calmar 区间 → 满仓持有 000852 → 当前 baseline `0.4103` → 无摩擦 NAV 择时上界 `0.4077`（标注 in-sample bias 方向）→ v3 门 `1.0`。回答：在该域内 Calmar > 1.0 需要什么数量级的结构改变（对冲 / 多空 / 跨域），long-only 域内是否存在已知可达路径。

### 2.4 备选门方案分析（只列选项 + 各自证据，不做推荐之外的裁决）

至少覆盖四个选项，每个给出"按该口径当前 baseline 与历史候选会怎样判定"的反事实回放（只读复算，不改任何 registry）：

- **A. 降低绝对 Calmar 阈值**：给出按指数对照与 baseline 分布推导的候选阈值（如同期最优指数 Calmar 的 1.5×，或 0.5 这类整数锚），列出该阈值下历史候选的判定变化；
- **B. 绝对门改超额口径**：absolute gate 的 Calmar 替换为已有 metric_definitions 中的 `excess_calmar_ratio`（vs 主基准 000852.SH），分析该指标的分母语义（同窗超额回撤）与可博弈性；
- **C. 双轨门**：绝对 Sharpe 门保留，Calmar 从 absolute gate 移到 relative gate 分支（实质 = 删除绝对 Calmar 硬门，靠既有 relative_gate 兜底），分析放权后的风险（纯 beta 策略是否会漏过——用 8 指数自身过门情况验证）；
- **D. 不改门**：保留 1.0 作为"北极星"，accepted 标准另行分级（research-accepted / production-accepted），分析对 OQ-010 路线优先级的含义。

每个选项必须回答：该口径下"满仓持有指数"会不会被误放行（门的反 beta 滥竽充数能力是底线）。

## 3. 预登记判读规则（先写后跑）

1. 若 8 指数同期 Calmar 全部 < 0.5 **且** 全部 3 年滚动窗口 Calmar < 1.0 **且** 无摩擦择时上界 < 0.5 → 报告结论为"**绝对 Calmar > 1.0 在当前域内物理不可达，门需要 owner 重新裁决**"，选项 A-D 进入 owner 决策。
2. 若存在指数或滚动窗口达到 1.0 量级 → 结论为"门可达但当前策略族距离远"，改策略优先、门暂不动。
3. 本 PRD 不修改契约：任何门变更须另立 contract v4 / 修订 PRD 并经 owner 批准（DECISION-20260608-12 冻结纪律）。
