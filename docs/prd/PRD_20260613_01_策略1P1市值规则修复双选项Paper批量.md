## 1. 背景与动机

Phase 0（matched official cost 主口径）已把 topdown 的唯一致命伤定位到 P1 规则集的**顺周期饱和**：

- T1−T0 CAGR gap = `-13.70pp`，四通道归因中现金差约 `-8.93pp/年`、持仓构成差约 `-4.53pp/年`，成本/换手与集中度通道近零——病根不是替换语义本身。
- 机制：崩盘后小市值股普遍触发 `total_mv_cny < 30e8` / `circ_mv_cny < 20e8` 两条市值规则，walk_depth 内无可替换标的。实证：2022-05-06 信号日 T1 top-50 P1 标记率 `98%`（市值规则 `96%`），随后 10 个交易日 T1 平均现金 `94.48%`，窗口 return gap `-9.07pp`——崩盘后满手现金踏空反弹。
- Phase 0 报告预登记的 owner 决策问题即本 PRD 两选项："是否剔除两条顺周期市值规则（保留崩盘形态规则），或为 P1 增加饱和回退"。

本 PRD 在同一 paper 框架上把两选项跑成可比证据，回答**topdown 是否值得进 Phase 2**——否则 Phase 2 的 Cloud Run 重跑是在未决问题上烧钱。

## 2. 实验设计

### 2.1 臂矩阵（在 Phase 0 既有 T0/T1 之上新增三臂）

P1 六条规则按 Phase 0 本地计算口径分解为两组：
**市值组** = `total_mv_cny < 30e8`、`circ_mv_cny < 20e8`；
**形态组** = `ret_20d < -30%`、`drawdown_20d < -30%`、`limit_down_days_20d >= 2`、`one_word_limit_days_20d >= 1`。

**required-field-NULL 分组（实现硬要求）**：当前脚本的 `tail_risk_reason()` 对六字段任一 NULL 统一返回 `tail_risk:required_field_null`，不可直接归组。实现必须改为生成**字段级** null reason（如 `tail_risk:null_total_mv_cny`），市值字段 NULL 归市值组、形态字段 NULL 归形态组；既有 T0/T1 臂的过滤行为不受该细化影响（T1 仍拦截全部 reason），细化只服务新臂的分组放行，单测须断言这一点。

| 臂 | 规则 | 说明 |
|---|---|---|
| T0 | 无 P1 | 既有对照臂（复跑保持同口径） |
| T1 | 完整六条 | 既有完整臂（复跑） |
| **T1a** | 仅形态组（剔除市值组） | 选项①：直接剔除两条市值规则 |
| **T1b1** | 完整六条 + 饱和回退到 T1a | 选项②变体 1：单次调仓内 walk_depth 候选 P1 标记率 > **0.60** 时，该次调仓放行市值组（退化为 T1a 规则） |
| **T1b2** | 完整六条 + 饱和回退到 T0 | 选项②变体 2：同阈值触发时该次调仓完全放行（退化为 T0） |

- 饱和阈值 `0.60` 沿用 Phase 0 既有 `p1_saturated_rebalance_count` 的定义（`p1_marked_rate > 0.60`），不引入新自由参数；阈值敏感性只作为附表（0.50 / 0.70 两档），不进入主判读。
- 标记率的分母与 Phase 0 同口径（当次调仓 walk_depth 内候选）；回退只影响**该次调仓**的新买入过滤，不追溯已持仓。

### 2.2 固定口径（与 Phase 0 完全一致，零新增自由度）

- 同一 prediction 流：run/backtest id 从记忆 resolver 解析（当前应解析到 true5y CA-on `..._v20260611_01` / `..._v20260611_01_ca01`），**禁止硬编码**。
- `walk_depth=50` 主判读（30 作为附表）；biweekly 调仓、PIT、可负担性含费判定、`initial_capital` 口径全部沿用 Phase 0。
- 成本三档沿用：matched official 分腿（买 `6bps` / 卖 `11bps`）为主判读，`0bps` / `20bps` 敏感性。
- Paper 局限声明沿用并补充：paper 为 raw 价格口径、不模拟分红送转与卖出失败降级；五臂共享同一口径，**臂间对比内部一致**，但绝对数字不可与 CA-on official ledger 直接互比（报告显式声明）。

### 2.3 输出

- 五臂 × 三成本档汇总矩阵（沿用 Phase 0 列契约：CAGR / vol / IR vs 000852 / MaxDD / Calmar / crunch excess / 平均现金 / 实现持仓数 / 集中度 / 换手 / P1 跳过计数 / 饱和调仓数 / 最大标记率）。

- 小 CSV（汇总矩阵）随 PR 入库；大明细 CSV 上传 GCS 由报告引用（沿用 Phase 0 / DOC_CONVENTIONS 口径）。

## 3. 预登记判读规则（先写后跑，review 重点）

以 matched official cost、walk_depth=50 为主判读：

1. **修复有效**：某修复臂同时满足 (a) 该臂−T0 CAGR gap 收敛到 > `-2pp`；(b) 饱和 episode 内平均现金 < `30%`（消除踏空）；(c) MaxDD 不比 T0 差超过 `+2pp`；(d) crunch 段（2024-01..02）超额不比 T1 差超过 `-3pp`（不完全丢掉 P1 的抗崩盘价值）→ 结论：**该臂规则带入 Phase 2**（多臂同时满足时取 CAGR gap 最小者；打平取规则更保守者 T1b1 > T1a > T1b2）。
2. **P1 不可救**：所有修复臂都不满足 1 → 结论：**建议 Phase 2 以 T0（无 P1）口径进入**，PRD_10 的 P1 绑定条款需修订（owner 决策项）。
3. **topdown 本身存疑（提示项，非硬判读）**：paper raw 口径与 CA-on official ledger 不可直接互比（§2.2 局限声明），因此 T0 绝对读数 vs 现行 baseline `0.4103` 的比较**只作提示**写入报告，不构成"整体缓做"判定依据；"topdown 整体缓做"结论只能由 Phase 2 真实 ledger（或 CA 调整后的 paper 证据）支撑，留 owner 决策。
4. 判读只回答"哪个口径进 Phase 2 / 是否进"，**不做 accepted / promotion / 默认 profile 变更**；最终 go/no-go 由 owner 拍板。
