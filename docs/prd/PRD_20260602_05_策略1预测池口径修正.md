> 文档维护：GPT-5（最近更新 2026-06-02）

> 范围声明：本文只修正策略 1 `ml_pv_clf_v0` 的 valid/test 预测池与评估口径，不处理 `12_qa_model_diagnosis_outputs.sql` 的 `split_tag` 歧义 bug，也不直接处理信号反向、标签重做、模型类型或组合参数。
> 关联：`PRD_20260601_01_策略1价格量价基础分类模型.md`、`PRD_20260601_02_策略1BQML回测闭环.md`、`PRD_20260602_04_策略1模型质量诊断.md`、`.agent/memory/OPEN_QUESTIONS.md` OQ-010。

## 1. 背景

策略 1 模型质量诊断已在当前基线 run 上跑通主流程：

```text
run_id = s1_bqml_20260601_01
backtest_id = bt_s1_bqml_20260601_01
```

诊断输出的主结论为：

```text
primary_diagnosis = sample_filter_risk
confidence = high
```

关键证据：

| 指标 | 当前值 |
|---|---:|
| `test_trainable_ratio` | 0.44 |
| `test_excluded_by_tradable` | 6,727 |
| `test_excluded_by_label_valid` | 8,261 |
| 平均不可解释排除率 | 49.81% |
| valid RankIC mean | -0.099 |
| test RankIC mean | -0.056 |

当前 runner 的数据路径是：

1. `sql/dws/06_dws_stock_sample_daily.sql` 定义 `sample_trainable_default`：
   ```text
   in_universe_default
   AND has_full_history_60d
   AND has_valuation_data
   AND label_entry_tradable
   AND label_valid_5d
   ```
2. `sql/ml/strategy1/01_build_training_panel.sql` 对 train/valid/test 全部使用 `s.sample_trainable_default` 写入 `ads_ml_training_panel_daily`。
3. `sql/ml/strategy1/04_predict_daily.sql` 只对 `ads_ml_training_panel_daily` 中 `split_tag IN ('valid','test')` 的行预测。

因此 valid/test 的预测池已经被 `label_entry_tradable`、`label_valid_5d` 这类未来字段过滤。训练阶段使用这些字段是合理的，因为训练需要标签；但 valid/test 预测阶段不能把未来可交易性或未来标签可得性作为入池条件，否则回测和诊断评估不再代表真实可用的 t 日预测场景。

## 2. 问题定义

### 2.1 当前问题

当前 valid/test prediction pool 不是 live-available prediction universe，而是 trainable labeled sample 的子集。

这带来三个问题：

1. **未来信息进入预测池选择**
   `label_entry_tradable` 依赖 t+1 开盘是否可买，`label_valid_5d` 依赖未来标签是否可计算。它们在 t 日预测时不可得。

2. **预测覆盖率被系统性压低**
   诊断显示 test trainable ratio 约 0.44，约一半 live 可观察股票没有进入预测池。模型分数、候选池、组合和回测都只在被未来条件过滤后的股票上运行。

3. **信号质量诊断被样本口径污染**
   当前 RankIC / bucket / TopN 收益同时反映模型信号和未来过滤后的样本选择，无法判断真实 live universe 上模型是否仍反向或弱。

### 2.2 不在本文处理的问题

以下问题必须拆开处理：

| 问题 | 是否纳入本文 | 处理方式 |
|---|---|---|
| `12_qa_model_diagnosis_outputs.sql` 的 `split_tag` 歧义 | 否 | 纯 QA bugfix，单独小 PR 直接修。 |
| valid/test 预测池依赖未来标签 / 可交易性过滤 | 是 | 本 PRD 主体。 |
| valid/test RankIC 为负、信号反向 | 否 | 等预测池口径修正并重跑诊断后再单独判断。 |
| 持股数、权重上限、调仓频率 | 否 | 组合参数实验，不能先于预测池口径修正。 |
| 新增财务、行业、资金面特征 | 否 | 后续特征实验，不属于本 PRD。 |

## 3. 目标

1. 将 train 样本池与 valid/test 预测池明确拆分：
   - train：使用 trainable labeled sample。
   - valid/test prediction：使用 t 日 live-available feature universe，不使用未来标签或未来可交易字段过滤。
2. valid/test 的模型预测、候选池、组合和回测必须从 live-available prediction pool 出发。
3. valid/test 的模型评价可以在事后 join 标签，但标签有效性只能用于评价子集，不得影响预测池入池和排名。
4. 产出 QA，证明 valid/test prediction pool 不依赖 `label_entry_tradable` / `label_valid_*` / `fwd_*` / `target_*` 等未来字段。
5. 重新跑 runner 和模型质量诊断，为后续 OQ-010 信号方向、标签、特征和组合参数实验提供干净基线。

## 4. 非目标

- 不保证本次改动后策略收益变好。
- 不直接反转 score。
- 不修改标签定义。
- 不修改 BQML 模型族。
- 不新增特征。
- 不调整持股数、单票权重上限、调仓频率或成本参数。
- 不重构 ADS 全部契约；P0 优先采用最小 schema 变更或无 schema 变更方案。

## 5. 核心口径

### 5.1 三类 mask

本文固化三类口径，后续实现必须显式命名或在 SQL 注释中写明。

| 名称 | 用途 | 允许字段 | 禁止字段 |
|---|---|---|---|
| `train_fit_mask` | 训练 BQML 模型 | t 日已知特征 + 事后标签有效性 | 无，训练允许使用标签有效性筛样本 |
| `predict_live_available_mask` | valid/test 预测入池、候选池、组合、回测 | t 日已知 universe / feature / history / valuation 掩码 | `label_entry_tradable`、`label_valid_*`、`fwd_*`、`target_label`、`target_return` |
| `eval_label_available_mask` | valid/test 事后模型质量评价 | 预测结果 + 事后标签有效性 | 不得反向影响预测池、rank、candidate、portfolio |

### 5.2 P0 默认定义

#### `train_fit_mask`

P0 沿用当前训练样本口径：

```text
sample_trainable_default = TRUE
```

即：

```text
in_universe_default
AND has_full_history_60d
AND has_valuation_data
AND label_entry_tradable
AND label_valid_5d
```

训练需要目标标签，使用标签有效性筛选是合理行为。

#### `predict_live_available_mask`

P0 定义为：

```text
in_universe_default
AND has_full_history_60d
AND COALESCE(has_valuation_data, FALSE)
```

说明：

- `in_universe_default` 是 t 日可知的股票池规则，包含板块、上市天数、ST、流动性、价格等默认过滤。
- `has_full_history_60d` 是 t 日特征完整性，live 可知。
- `has_valuation_data` 是 t 日估值特征可得性，live 可知。
- 不得加入 `label_entry_tradable`、`label_valid_5d` 或任何 forward return / label 字段。

如果后续有特征列允许缺失并由 BQML 处理，不能因为标签缺失剔除预测行；如果某个特征必须非空，应在 `predict_live_available_mask` 中以 t 日特征完整性字段表达。

#### `eval_label_available_mask`

P0 评价主 horizon 仍为 5d，事后评价子集定义为：

```text
prediction row exists
AND label_valid_5d = TRUE
AND fwd_xs_ret_5d IS NOT NULL
AND label_top30_5d IS NOT NULL
```

说明：

- 该 mask 只能用于 RankIC、bucket、AUC、TopN 事后评价。
- 该 mask 不得用于 `ML.PREDICT` 输入。
- 评价报告必须同时披露：
  - full prediction rows；
  - label-available evaluation rows；
  - evaluation coverage ratio。

## 6. 功能需求

### FR-LIVE-1 拆分 train 与 valid/test 入池逻辑

`sql/ml/strategy1/01_build_training_panel.sql` 必须按 `split_tag` 使用不同入池 mask。

P0 允许继续写入同一张 `ads_ml_training_panel_daily`，但语义需从“纯训练样本表”收敛为“run-scoped model input panel”：

| split | 入池规则 | `target_label` / `target_return` |
|---|---|---|
| train | `train_fit_mask` | 必须非空 |
| valid | `predict_live_available_mask` | 可为空；有标签时用于事后评价 |
| test | `predict_live_available_mask` | 可为空；有标签时用于事后评价 |

实现时可以从 `dws_stock_sample_daily` 读取特征和可选标签，但 WHERE 条件必须 split-aware：

```sql
WHERE s.trade_date BETWEEN p_train_start AND p_test_end
  AND s.feature_version = p_feature_version
  AND s.label_version = p_label_version
  AND (
    (s.trade_date BETWEEN p_train_start AND p_train_end AND s.sample_trainable_default)
    OR
    (s.trade_date BETWEEN p_valid_start AND p_test_end
      AND s.in_universe_default
      AND s.has_full_history_60d
      AND COALESCE(s.has_valuation_data, FALSE))
  )
```

### FR-LIVE-2 valid/test `ML.PREDICT` 输入不得使用标签过滤

`sql/ml/strategy1/04_predict_daily.sql` 的预测输入可以继续来自 `ads_ml_training_panel_daily`，但必须保证：

- valid/test panel 已按 `predict_live_available_mask` 构建；
- `ML.PREDICT` 输入 SELECT 不引用 `target_label` / `target_return`；
- `ML.PREDICT` 输入 WHERE 不引用 `label_entry_tradable`、`label_valid_*`、`sample_trainable_default`、`fwd_*`。

### FR-LIVE-3 valid 选型要区分预测全集和评价子集

`sql/ml/strategy1/03_select_model_and_register.sql` 当前 valid 候选预测和 `ML.EVALUATE` 都来自 `ads_ml_training_panel_daily split_tag='valid'`。

修正后：

1. candidate prediction 必须对 valid full prediction pool 运行。
2. RankIC / bucket / TopN 等收益评价必须将 candidate predictions 事后 join 标签，并只在 `eval_label_available_mask` 上计算。
3. AUC / logloss 可以继续使用 `ML.EVALUATE`，但输入必须过滤到 `target_label IS NOT NULL` 或等价的 label-available subset。
4. registry `metrics_json` 必须新增或保留可解析字段：
   - `valid_prediction_rows`
   - `valid_eval_rows`
   - `valid_eval_coverage`
   - `valid_trainable_legacy_rows` 或等价 legacy 对照计数

模型选择仍以 valid RankIC 为主，但该 RankIC 必须来自 full prediction pool 的事后 label-available subset，而不是先用 label-valid 过滤出的预测池。

### FR-LIVE-4 05-08 下游必须从修正后的预测池继承

`05_build_candidates.sql`、`06_build_portfolio_targets.sql`、`07_build_order_plan.sql`、`08_run_backtest.sql` 不应新增未来标签过滤。

要求：

- 05 候选池继续从 `ads_model_prediction_daily` 出发。
- 05 可用 t 日 `dws_stock_universe_daily.in_universe_default` 二次确认股票池，但不得 join `dws_stock_label_daily` 做入选过滤。
- 08 回测继续用实际 t+1 可交易状态撮合，买不到就 skip/carry；这属于执行仿真，不属于预测池过滤。

### FR-LIVE-5 诊断必须能识别口径是否修正

`scripts/strategy1/diagnose_model_quality.py` 和 `sql/ml/strategy1/11_model_quality_diagnostics.sql` 必须输出：

- valid/test full prediction rows；
- valid/test label-available eval rows；
- valid/test legacy trainable rows；
- `live_prediction_coverage_ratio`；
- `eval_label_coverage_ratio`；
- 是否检测到 valid/test prediction pool 仍由 `sample_trainable_default` 过滤。

如果 valid/test prediction rows 与 legacy trainable rows 基本相等，且诊断发现所有预测行均满足 `sample_trainable_default`，则仍应输出 `sample_filter_risk`。

### FR-LIVE-6 QA 门禁

需要新增或修改 QA，至少覆盖：

1. `12_qa_model_diagnosis_outputs.sql` 的 `split_tag` 歧义 bug 单独修复，保证诊断 QA 可执行。
2. valid/test prediction panel 行数必须大于等于 legacy trainable 行数。
3. 当前基线重跑后，valid/test prediction panel 应包含一部分 `sample_trainable_default=FALSE` 但 `predict_live_available_mask=TRUE` 的行；若为 0，QA 应失败或至少给出人工确认项。
4. train split 中 `target_label` 和 `target_return` 不得为空。
5. valid/test `ads_model_prediction_daily` 行数必须等于 selected model 在 valid/test full prediction panel 上的预测行数。
6. 04/05/08 不得通过 label 字段过滤 candidate / portfolio / backtest 输入；可用 SQL 文本检查 + reviewer 人工项，或 BigQuery 结果断言间接覆盖。
7. 诊断 summary 中 `sample_filter_risk` 不得再由 live-unavailable prediction pool 触发；若仍触发，必须有新的 evidence 字段说明原因。

## 9. 验收标准

### 9.1 口径验收

- train rows 全部满足 `sample_trainable_default=TRUE`。
- valid/test prediction rows 全部满足 `predict_live_available_mask=TRUE`。
- valid/test prediction rows 不要求满足 `label_entry_tradable` 或 `label_valid_5d`。
- 当前基线重跑后，valid/test prediction rows 数量应明显大于 legacy trainable rows，并报告覆盖率。
- valid/test rank 和 candidate 由 full prediction pool 生成；label-valid subset 只用于事后评价。

### 9.2 运行验收

- runner 01-10 端到端通过。
- `render_report.py` local 或 uploaded 模式通过。
- `diagnose_model_quality.py` local smoke 和 uploaded 模式通过。
- `12_qa_model_diagnosis_outputs.sql` 通过。
- GCS / ADS artifact 状态正确。

### 9.3 诊断验收

实现后重新诊断，必须能回答：

1. `sample_filter_risk` 是否仍存在？
2. 如果不存在，valid/test RankIC 是否仍为负？
3. full prediction pool 与 label-available eval subset 的 coverage 分别是多少？
4. 策略差是否从“样本池口径问题”转移为“信号方向 / 标签 / 特征 / 组合”问题？

注意：本文不要求 RankIC 转正，也不要求 NAV 改善。本文只要求评估口径正确。

## 10. 风险与取舍

### 10.1 valid/test 样本扩大后指标可能更差

这是预期可能结果。当前指标来自未来过滤后的子集，不可作为 live 评估口径。口径修正后，RankIC、AUC、回测收益都可能下降，但可信度更高。

### 10.2 label-available coverage 下降

valid/test full prediction pool 扩大后，有标签评价覆盖率可能下降。必须报告 coverage，不能只展示 label-available subset 上的好指标。

### 10.3 ADS 表名历史遗留

`ads_ml_training_panel_daily` 名称含 training，但 P0 可继续复用为 run-scoped model input panel。若后续容易误解，可单独 PRD/PR 引入 `ads_ml_model_input_panel_daily` 或 `ads_model_input_panel_daily`，但不作为本文 P0 必需项。

## 11. 后续决策门槛

本 PRD 实现并重跑诊断后：

- 如果 `sample_filter_risk` 关闭且 RankIC 仍稳定为负，下一步才进入信号方向 / 标签 / 特征 PRD。
- 如果 RankIC 接近 0 且 bucket 无单调性，优先做特征和标签实验，不优先调组合参数。
- 如果 RankIC 有正信息但 NAV 仍差，再进入组合参数实验：调仓频率、持股数、单票权重上限、成交约束。
- 如果样本池修正后仍出现 live-unavailable 过滤，必须继续修数据口径，不能进入策略调参。
