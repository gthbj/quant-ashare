## 1. 假设与目标

### 1.1 绑定约束诊断
见 §0.3：v1 长窗 0.6685 = long-only × 偏小盘选股；alpha 在大盘存活且**价值因子纯多头可采**。暴露管理（空仓/vol-target）已被 PR #222 证伪为 Sharpe 杠杆，故本 PRD 的 Sharpe 抬升全部押在 alpha/组合构造侧。

### 1.2 核心假设（机制链，注明证据强度）
把以下三者叠加，可把长窗 contract Sharpe 往 1.5 推：
1. **size 重心搬迁**（方向有探针支撑、绝对幅度无保证）：探针证实纯多头超额 Sharpe **方向上**随市值单调上升，把组合 size 重心从小盘搬到大盘理论上提高 long-only 可实现 Sharpe；不靠对冲（探针证实对冲 beta 无用）。⚠ 0.59/0.20/0.09 是 in-sample 零成本 composite 下界、无显著性区间，仅支撑方向，**不作为可实现 Sharpe 锚点**。
2. **alpha 质量切换**：价值因子在大小盘等强（ep_ttm/bp/dividend_yield_ttm），把 edge 来源从「小盘反转（空头侧采不到）」换成「大盘价值（多头侧可采）」，提高 long-only 可实现 IR；20d horizon 对齐到探针中大盘纯多头超额 Sharpe 最高的窗口。
3. **目标对齐（机制已按 Codex round-1 校正、强度下调）**：**主驱动在面板层**——`label_horizon=20` + 价值倾斜特征 + size-aware 样本权重改变「每个候选学什么」。**选模型 reducer 层不是 P0 自动偏向头部的**：年度滚动选模型在 `select_register_predict.py:382-480`，主排序是 eligible / CV rank IC / CV spread / valid IC / valid spread，`valid_topn_fwd_ret_mean` 只是**后置排序项**；因此把 `evaluate_scores` 的 `topn` 对齐到 `target_holdings` 只是**口径一致性修复**，**不足以**让选模型重心移到 top-K 头部。是否进一步把 reducer 主排序/eligible 规则改成头部捕获导向，是**显式开放决策**（§10），不在 P0 默认宣称。

### 1.3 目标与 1.5 的诚实标注（钉死，避免窗口/口径误导）
- **主判据 = 长窗 contract Sharpe**（窗 `2021-01-04..2026-06-09`，CA-on，`ledger_exec_v1_lot100`）。1.5 ≈ v1 长窗 0.6685 的 **2.24×**，是高 bar。
- **近窗（`2024-01-02..2026-04-30`）v1 已 1.4636**（来源 `docs/分析-策略1v3Calmar门合理性-20260613.md:56`；另有探针/会话锚点 1.4642，二者差约 0.0006；**本 PRD 统一引用 1.4636 并注明来源**）。**近窗达 1.5 不构成长窗达标证据**：PR #222 已证近窗 baseline 无 overlay 已 **daily 1.337 / contract 1.4168（≈1.46）**、是**满仓子区间属性**，且长窗含 2021-2022 大盘价值相对成长持续跑输的熊市段，是真正考验。
- 一律用 **contract Sharpe**（复合年化/年化波动，rf=0，复用 `replay_acceptance_gate_v3.py` helper），**不引用 legacy summary Sharpe**（legacy 0.6834 / effective-window contract 0.6076 不可混引，`KNOWN_CONSTRAINTS.md:114`）。
- **诚实降级路径（预登记）**：1.5 long-only 高于探针自身的 1.2 判断且探针认为 long-only 连 1.2 都大概率需要空头腿。因此：
  - 若 P0 主 arm 长窗 contract Sharpe **收敛在 0.9–1.2 区间**，即视为「大盘价值倾斜机制证实、long-only 上限逼近」，**是否解禁大盘融券空头腿（脱离本 PRD 的 long-only 约束）留 owner 决策**，不在本 PRD 内做空；
  - 失败/停做判据见 §5。

## 2. Alpha 引擎设计

> 设计目标：用**最小必要改动面**把组合系统性推向大盘价值头部，**P0 不触 ledger/portfolio/order SQL、不改任何默认、不改既有 feature set 列定义**。但 P0 **不是「零/小改」**——按 Codex round-1 核实，`label_horizon` 与 `weight_version` 必须作为一等参数贯穿编排/Experiment/窗口/CV embargo/synthetic continuous/catalog/manifest/测试，整体是一个**有界的「中改」工程包**。下文逐杠杆诚实分级。

### 2.1 训练目标 / objective（偏向 long-only 头部捕获）

**交易域不变**：标签横截面派生口径 = 当日 `in_universe_default_for_rank` 内（沪深主板，`sql/dws/04_dws_stock_label_daily.sql:169-172`），是「主板内超额」，符合约束、不扩域。

**v1 现状**：回归目标 = `ads_ml_training_panel_daily.target_return` = `fwd_xs_ret_5d`（横截面去均值超额，连续值；`label_version='open_to_close_h1_5_10_20_v20260601'`，标签口径 t→t+1 开盘入场、t+5 收盘退出、后复权）；`model_family=lightgbm_regression`。

### 2.2 价值倾斜特征

> ⚠ **内部 critique 核心修正（feasibility-1）**：`build_training_panel_risk_feature.sql` 的 `TO_JSON_STRING(STRUCT(...))`（`:202-281`）与 `feature_column_list` ARRAY_CONCAT（`:283-314`）**不按 `feature_set_id` 裁列**——整段 SQL **只能产出 3 种固定列集**（PV / PV+fin / PV+fin+risk），由 `p_fin_enabled`/`p_risk_enabled` 两个布尔整块开关。要产出「剔除短反转/低波/换手」的**任意裁剪子集**，**必须新建独立 panel step 或在现 step 加 feature_set 驱动的条件分支 + 新布尔 gate**，并在 catalog 登记。这是**「较大改 SQL」**，列序三处一致校验（`feature_sets.py` 列 list / STRUCT / ARRAY_CONCAT vs `prepare_matrix.py:158/164-168` 强校验）是头号实现风险。

因此分阶段：

## 3. 稳健层（long-only 之上的回撤层，**明确不计入 Sharpe 达标**）

**定位（PR #222 已三路交叉验证为实测事实）**：「适当空仓 / 降仓 / vol-target」是**稳健/Calmar 层、不是 Sharpe 杠杆**（`scripts/strategy1/simulate_cash_overlay_sharpe_contribution.py`、`docs/分析-策略1空仓降仓overlay对Sharpe贡献-20260613.md`、`docs/analysis_strategy1_cash_overlay_sharpe_20260613_results.csv`，merge `ecc811c`）：
- **vol-targeting（教科书 Sharpe 杠杆）失效**：8 变体长窗 **daily** Sharpe 全 ≤ baseline（daily 0.42–0.67）、Calmar 全降。波动率不是本 alpha 坏收益的前瞻量。→ **本 PRD 据此把「连续 vol-target」从可选 Sharpe 路径正式降级为「已证伪的 Sharpe 杠杆，至多 Calmar 层」，不再作为抬 Sharpe 的待实现方向。**
- 唯一能抬 Sharpe 的是**回撤动量降仓，但小且脆弱**：零成本 +0.058 daily / +0.073 contract，20bps 后缩到 +0.02/+0.03、contract 跌破 0.70 门，且是 in-sample best-of-grid 上界。
- **market-state 降仓是纯回撤层**：MaxDD −37.4%→−20.4%、Calmar→0.52，但 Sharpe 反降。
- 先前 calmar-gate 仿真给的 `two_state_biweekly_elow0` 无摩擦 **contract** Sharpe 0.6685→0.7478（`docs/analysis_strategy1_v3_calmar_gate_20260613_exposure_overlay.csv`，in-sample best-of-48、零成本）是**乐观上界**，PR #222 已证扣成本后该类微弱抬升脆弱、不可实现。
- **口径边界**：上述条件于 long-only/exposure≤1/零成本/粗粒度 overlay，非普适；「小且脆弱」是定性判断（未做 paired-Sharpe 显著性检验）。

**结论**：稳健层只押 MaxDD/Calmar，**1.5 Sharpe 必须来自 §2 的 alpha/组合构造**。三档稳健层（research-only，**默认 `tail_risk_profile_id=diagnostic_only` 不变**，`KNOWN_CONSTRAINTS.md:127/134`）：
1. **现成 skip-new-buys 空仓**（零新 SQL）：`tail_risk_profile_id='market_risk_off_v0'`（`ledger.py:24-29`）。⚠ **Codex round-1 High-6 行号修正**：跑 bottom-up **`ledger_exec_v1_lot100`** 时 market risk-off 的判定/执行在 `ledger.py:413-417` 与 `ledger.py:840-849`（risk-off 信号日次一开市日对未持仓新目标写 `BUY_SKIPPED_MARKET_RISK_OFF`、不强卖）；**之前误引的 `ledger.py:542-551` 属 `build_daily_plan_topdown` 分支，不是 v1 路径**。读取信号集仍在 `ledger.py:1313-1319`。
2. **入选不足保留现金**（现成）：`build_portfolio_targets.sql:47` 等权 `LEAST(SAFE_DIVIDE(1.0, p_target_holdings), p_max_single_weight)`，入选不足 N 只剩余现金自然保留。
3. **连续 vol-target**：**已被 PR #222 证伪为 Sharpe 杠杆，本 PRD 不再立项**（如未来仅作 Calmar 层需 owner 单独决策；ledger 层尚无实现，属新写代码）。

- **选型硬约束**：含市场层 risk-off 的稳健层**只能跑 `ledger_exec_v1_lot100`**；**`ledger_exec_v2_lot100_topdown` 由 `is_topdown_tail_risk_profile_allowed`（`ledger.py:1603-1604,1711-1712`）禁止 `market_risk_off_v0`**（否则 raise ValueError）。
- QA 复用 `sql/strategy1/qa/qa_tail_risk_overlay_ab_outputs.sql`（arm→profile 映射 baseline=diagnostic_only/A2=market_risk_off_v0，guard 断言）。

- 指标公式一律复用 `replay_acceptance_gate_v3.py` helper（`compound_annualized_return`=`(end/start)^(252/(N-1))-1`、`annualized_volatility_from_daily_returns`=`std_samp*sqrt(252)` ddof=1、`max_drawdown_window`、`signed_zero_safe_ratio`，rf=0），**不得自定义本地指标**（metric freeze）。
- ⚠ **model_quality_parity caveat（Codex round-1 Medium-1）**：`annual_rolling_lgbm_regression_v0.yml:35` 固定 `bqml_reference_run_id=s1_bqml_baseline_pvfq_n30_bw_h5...`，`train_predict.py:774-783/792-793` 直接比 `valid_topn_fwd_ret_mean`。改 topn=20 / horizon=20 后 parity 会变成**混口径**。实现须二选一：为 topn/horizon 写入 metric metadata 并对同口径比较，或对本 research line **禁用/降级 parity**（`model_quality_parity_status` 不作判读依据）。

### 5.4 预登记判读（钉死，避免 in-sample selection bias）
- **达标判据**：「达到/接近 1.5」钉在**长窗 contract Sharpe**；报告同时披露近窗读数与窗口敏感性。
- **可证伪失败/停做判据（必须，critique eval-2）**：**P0 先验主 arm = 20d 回归 + 市值 sample_weight + 现成 set**。若主 arm 长窗 contract Sharpe **相对 v1 0.6685 提升 < 绝对 +0.10**（即长窗 < ~0.77），即判「size/价值/目标对齐三链条机制未兑现」，**stop 并报告**，不继续堆 arm。
- **降级判读**：长窗 Sharpe 收敛在 0.9–1.2 → 机制证实、long-only 上限逼近，是否解禁空头腿留 owner（§1.3）。
- **arm 选择规则预登记**：主 arm 为 20d 回归 + 市值权重；`weight_version`（映射形式 + `w_max ∈ {2,3}` 网格）、价值倾斜列集（Phase 1.5）、holdings(n20/n10)、对照 arm（头部加权 / 分类 / fin 变体 / reducer 头部排序）**全部预登记为对照实验，不得事后顶替主结论**。
- **覆盖率混杂复核（critique honesty-3）**：报告须在等覆盖率 / `tree_winsor_missing_passthrough_v1` missing-passthrough 口径下复核大盘价值优势，不把覆盖率差异误读为 alpha 差异。
- **稳健层不计入 Sharpe 判读**：PR #222 已证空仓/降仓不是 Sharpe 杠杆；稳健层只在 Calmar/MaxDD 维度作旁证，先跑无摩擦上界（`simulate_cash_overlay_sharpe_contribution.py` / `simulate_exposure_overlay_upper_bound.py`）再决定是否投真实 ledger，结果标为机制天花板。

## 6. 成本 / 容量 / 运行预算

- **P0 实验规模上界**：主 arm 1 个 feature set × `label_horizon=20` × `weight_version∈{constant_1p0_v0(对照v1), logmv_xs_monotone_v0}` × holdings∈{n20,n10} ≈ **≤4 条年度滚动 lineage**。**每条 lineage 的真实训练量（Codex round-1 Medium-2 修正）= 6 个 backtest_year（2021–2026），每年 = selection 阶段 11 个候选 fan-out + 对 selected candidate 做 1 个 final-refit**（`annual_rolling_plan.py:264-303`），即每条 lineage ≈ 6×(11 selection + 1 final-refit) + 1 个 synthetic continuous 合成 + 1 个 CA-on continuous backtest。对照 arm（头部加权 / fin 变体 / reducer 头部排序 / 稳健层 A/B）显式列为追加，须 owner 批准后再跑。
- **并发口径**：每年 11 候选 fan-out 沿用 `candidate_count=11 / candidate_parallelism=11`、单 task `2 vCPU / 8Gi`（`annual_rolling_lgbm_regression_v0.yml:25-29`）。多实验 orchestrator 的 `--max-parallel-experiments` 须 owner 显式设；未设不得隐式降并发（`KNOWN_CONSTRAINTS` Cloud Run orchestrator 约束）。
- **预算原则**：本 PRD 只写方案；真实 Cloud Run run 数量、镜像重建、运行成本须在实现 PR 阶段按上界向 owner 报批，不在 PRD 阶段触发任何 live 执行。
