## 1. 背景

OQ-004 的核心问题是：策略、DWS/ADS 和 runner 不能按“常见指数代码”假设基准存在，而必须以 BigQuery ODS `index_daily` 实际存在的 Tushare `ts_code` 端点为准。

当前已知风险：

- ODS 当前没有 `index_daily_000300_SH`，但有 `index_daily_399300_SZ`；沪深300必须通过 `399300.SZ -> 000300.SH` 输出 canonical 代码。
- 中证1000 `000852.SH` 有 `index_daily` 价格端点，但没有 `index_dailybasic` 估值/市值端点。
- 中证2000、国证2000等候选基准，在当前 ODS `index_daily` 中未见可用端点，不能直接写成默认基准或 QA 必须存在项。
- runner 如果只校验基准代码“看起来存在”，仍可能在回测窗口内遇到缺失、重复或关键价格为空，导致 NAV / 超额收益不可信。

本文将 PR #10 的最终范围收窄为 OQ-004。财务报表口径事项已由独立 PRD 承载，不属于本文交付范围。

## 5. 功能需求

### FR-IDX-1：新增 canonical 指数映射维表

建议新增 `ashare_dim.dim_index`，至少包含：

```text
sec_code                 -- canonical 指数代码
source_sec_code          -- ODS/Tushare 实际 ts_code
index_alias
index_name
index_family
daily_endpoint
dailybasic_endpoint
daily_first_trade_date
daily_last_trade_date
dailybasic_first_trade_date
dailybasic_last_trade_date
has_daily
has_dailybasic
is_benchmark_candidate
benchmark_note
source_system
ingested_at
```

首版 `dim_index` 可以由静态候选清单 + ODS 实测端点聚合生成；未来可接入更完整指数主数据。

### FR-IDX-2：`dwd_index_eod` 使用映射维表

`dwd_index_eod` 必须继续输出：

```text
sec_code          -- canonical 代码
source_sec_code   -- ODS 实际代码
index_alias
```

并从 `dim_index` 或等价映射源驱动可用端点清单。现有 `399300.SZ -> 000300.SH` 规则必须保留。

### FR-IDX-3：区分收益基准与估值特征可用性

基准收益只要求 `has_daily = TRUE`。

指数估值/市场状态特征若需要 `pe` / `pb` / `total_mv_cny` 等字段，必须要求 `has_dailybasic = TRUE`。

因此：

- `000852.SH` 可用于中证1000收益基准。
- `000852.SH` 不可作为依赖 dailybasic 估值字段的市场状态来源，除非上游补齐 dailybasic endpoint。

### FR-IDX-4：runner 必须校验 benchmark 窗口覆盖

策略 runner 接受 `benchmark_sec_code` 时，必须在执行前对**完整 NAV / 超额收益写入窗口**做可执行数据契约校验。首版窗口至少覆盖 `test_start_date` 至 `test_end_date`；如果 runner 使用独立 `backtest_start_date` / `backtest_end_date`，则以回测 NAV 实际写入窗口为准。

校验基准为 `dim_trade_calendar` 中 `exchange='SSE' AND is_open=1` 的开市日，要求：

1. `benchmark_sec_code` 存在于 `dwd_index_eod.sec_code`。
2. 对窗口内每个开市日，`dwd_index_eod` 必须有且只有一条记录满足 `sec_code = benchmark_sec_code`。
3. 每条基准记录的 `close` 非空；用于计算日收益的 `pre_close` 或等价可计算字段非空。若实现选择 `LAG(close)` 计算日收益，则要求窗口首日之前至少存在一个可用开市日基准记录。
4. `MIN(trade_date) <= window_start` 且 `MAX(trade_date) >= window_end`。
5. 任一开市日缺失、重复、关键价格为空，或回测起点早于指数可用起点，runner 直接失败并输出可读错误。

推荐实现为 runner 前置断言或独立校验 CTE：

```sql
WITH calendar AS (
  SELECT cal_date AS trade_date
  FROM `data-aquarium.ashare_dim.dim_trade_calendar`
  WHERE exchange = 'SSE'
    AND is_open = 1
    AND cal_date BETWEEN window_start AND window_end
),
benchmark AS (
  SELECT
    trade_date,
    COUNT(*) AS row_count,
    COUNTIF(close IS NULL OR pre_close IS NULL) AS bad_price_count
  FROM `data-aquarium.ashare_dwd.dwd_index_eod`
  WHERE sec_code = benchmark_sec_code
    AND trade_date BETWEEN window_start AND window_end
  GROUP BY trade_date
)
SELECT
  COUNTIF(b.trade_date IS NULL) AS missing_open_days,
  COUNTIF(b.row_count != 1) AS duplicate_days,
  COUNTIF(b.bad_price_count > 0) AS bad_price_days
FROM calendar AS c
LEFT JOIN benchmark AS b USING (trade_date);
```

上述三项均必须为 0；若首日收益依赖 `LAG(close)`，还必须额外校验 `window_start` 前一个开市日存在非空 `close`。

### FR-IDX-5：不可用端点不得硬编码进业务逻辑

中证2000、国证2000等当前未在 ODS `index_daily` 中出现的指数，只能记录为“候选待补采”，不得进入：

- DWD 读取 endpoint 列表
- DWS/ADS benchmark 默认值
- runner 示例配置
- QA 必须存在断言

## 6. QA 与验收

新增 OQ-004 指数断言：

- `dim_index` 中 `source_sec_code` 唯一。
- `dim_index` 中 `sec_code` + `source_sec_code` 映射唯一。
- `399300.SZ` 必须映射到 `000300.SH`。
- `dwd_index_eod` 中 `sec_code='000300.SH'` 时 `source_sec_code='399300.SZ'`。
- `000852.SH` 在 2019+ 有价格行；`has_dailybasic=FALSE`。
- 未在 ODS 出现的候选指数不能被标记为 `is_benchmark_candidate=TRUE`。
- runner 的 `benchmark_sec_code` 示例值必须通过 FR-IDX-4 的完整窗口覆盖契约：窗口内每个开市日有且只有一条基准记录，关键价格非空，无缺失、重复或起止日期不足。

关闭 OQ-004 的最低条件：

1. 当前可用指数端点、canonical 映射、起止日期已物化到 `dim_index` 或等价维表。
2. `dwd_index_eod` 从该映射读取并通过 QA。
3. runner / PRD 不再把未验证端点作为默认基准。

## 8. 风险与处理

| 风险 | 影响 | 处理 |
|---|---|---|
| `dim_index` 静态清单与 ODS 实际端点漂移 | runner 使用不可用 benchmark | 每次重建 `dim_index` 用 ODS 实测端点刷新起止日期和可用性。 |
| 中证1000无 dailybasic | 市场状态估值字段为空 | 收益基准与估值特征可用性分开标记。 |
| 基准窗口不覆盖回测区间 | 超额收益/NAV 不可信 | runner 前置失败，不做隐式填充。 |
| 候选指数被误设为默认基准 | 策略报告不可复现 | QA 禁止未见 ODS `index_daily` 的指数进入默认配置。 |

## 9. 实现处理结果

1. 策略 1 runner 示例基准保留 `000852.SH`。它满足收益基准所需的 `has_daily=TRUE`，但 `has_dailybasic=FALSE`，因此不能作为依赖指数估值字段的市场状态来源。
2. 新增 `dim_index` 作为指数 canonical 映射、端点可用性和 benchmark 候选状态的长期承载表。
3. OQ-004 以 `dim_index`、映射驱动的 `dwd_index_eod`、`sql/qa/03_oq004_index_checks.sql` 和 runner benchmark 窗口契约为关闭条件。
